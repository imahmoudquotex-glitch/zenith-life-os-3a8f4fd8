# المرحلة 43 — الإضافات الشاملة القابلة للتركيب (LifeOS Sticky Modules)

## 🧠 توسعة تنفيذية ضخمة — Modular LifeOS Add-ons Operating System

المرحلة 43 لازم تبقى طبقة **Modules Platform** كاملة، مش مجرد مجموعة صفحات منفصلة. كل Module يجب أن يشتغل كمنتج صغير مستقل: له schema، routes، widgets، AI tools، permissions، import/export، settings، analytics، وintegration واضح مع باقي النظام. الهدف إن المستخدم يلاقي داخل LifeOS كل الأدوات اليومية بدون فتح تطبيقات أخرى، لكن من غير تكرار مصادر الحقيقة الموجودة في مراحل سابقة.

<aside>
🧩

تعليمات للـ AI/Claude Code: لا تبني كل Module كصفحة عشوائية. ابنِ Module Registry أولًا، ثم Module Contract، ثم Data Models، ثم UI Surfaces، ثم AI Tools، ثم Tests. أي Module لا يلتزم بالـ contract لا يدخل الـ dashboard.

</aside>

### فلسفة المرحلة

- **Composable Modules:** كل module قابل للتحميل lazy-load، وله widget صغير، وصفحة كاملة، وquick capture action.
- **Single Source of Truth:** لا تعيد بناء tasks/goals/finance لو موجودة قبل كده؛ استخدم read models أو adapters.
- **Privacy by Design:** mood/journal/finance/health لا تدخل AI إلا بموافقة صريحة وبملخصات آمنة.
- **Desktop-first:** keyboard shortcuts، command palette، quick capture، split views.
- **Dark-only Native:** كل module له UI غني داخل dark mode بدون تباين ضعيف.
- **Performance Budget:** module غير مفتوح لا يُحمّل bundle بتاعه.

### ممنوعات صارمة

- ممنوع Module يقرأ Vault.
- ممنوع Finance يستخدم `float`, `numeric`, `decimal` للمال؛ استخدم `*_cents BIGINT`.
- ممنوع Journal/Mood يدخل AI raw بدون consent.
- ممنوع تحميل كل الـ modules في dashboard دفعة واحدة.
- ممنوع Module يعمل notification خارج Notification Engine.
- ممنوع أي paywall أو “unlock”.
- ممنوع تكرار goals/tasks truth tables بدون migration صريحة.

## ⚡ Focus Module تنفيذ أعمق

```tsx
export async function startFocusSession(ctx: RequestContext, input: {
  taskId?: string
  plannedMinutes: number
  soundPack?: string
  blockedUrls?: string[]
  mode: 'pomodoro' | 'deep_work' | 'flow'
}) {
  const session = await focusRepo.create({
    workspaceId: ctx.workspaceId,
    userId: ctx.userId,
    taskId: input.taskId,
    plannedMin: input.plannedMinutes,
    soundPack: input.soundPack,
    blockedUrls: input.blockedUrls ?? [],
    mode: input.mode,
  })

  await notificationEngine.suppressNonCritical(ctx.userId, {
    reason: 'focus_session',
    until: addMinutes(new Date(), input.plannedMinutes),
  })

  await moduleEvents.record(ctx, 'focus', 'focus.started', session.id)
  return session
}
```

## 🧠 Master Architect Upgrade — Sticky Modules Boundary Gate

- **Canonical module ownership:** لا تعيد إنشاء Goals/Finance/Tasks truth لو موجودة في W10/W11/W38. W40 يضيف UI/read models/integrations فقط أو migration صريحة.
- **Money invariant:** finance tables تستخدم `*_cents BIGINT` وFX bps، ولا `numeric/float` للمال. workout weights ممكن numeric لأنها قياس وليست money.
- **AI context minimization:** journal/mood/sleep/finance تدخل AI كملخصات consent-based، مع DLP؛ vault ممنوع تمامًا.
- **Health disclaimer:** mood/sleep/workout insights ليست نصيحة طبية. استخدم wording داعم، opt-out، وحذف كامل للبيانات.
- **Desktop shortcuts:** كل module له quick capture/command palette action، lazy-loaded bundle، وdashboard widget لا يكسر shell performance.

### 5) Finance + budget engine

```tsx
export async function monthSpend(userId: string, month: string, category?: string) {
  return db.querySql(
    `select coalesce(sum(amount_cents),0) as spent
     from finance_transactions
     where user_id=$1 and to_char(date,'YYYY-MM')=$2 and is_income=false
     ${category ? 'and category=$3' : ''}`,
    category ? [userId, month, category] : [userId, month])
}
export async function budgetStatus(userId: string) {
  const budgets = await db.finance_budgets.where({ user_id: userId })
  const month = new Date().toISOString().slice(0,7)
  const rows = await Promise.all(budgets.map(async b => {
    const spent = (await monthSpend(userId, month, b.category))[0].spent
    return { ...b, spent, pct: Math.round(spent / b.limit_cents * 100), over: spent > b.limit_cents }
  }))
  return rows
}
```

export function scoreMatrix(criteria: Array<{name:string;weight:number}>, options: Array<{name:string;scores:Record<string,number>}>) {
  return options.map(o => ({
    name: o.name,
    total: criteria.reduce((s,c) => s + (o.scores[c.name] ?? 0) * c.weight, 0),
  })).sort((a,b) => b.total - a.total)
}
