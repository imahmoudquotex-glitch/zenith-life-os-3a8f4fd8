## 🧠 Master Architect Upgrade — SRE Foundation Gate

- **SLO هرمي:** كل خدمة عندها SLI (Latency/Availability/Saturation/Errors) → SLO رقمي → Error Budget شهري → automated burn-rate alerts.
- **Distributed tracing:** OTel SDK في كل process (web/api/workers/realtime). Jaeger/Tempo backend. trace_id يتنقل عبر كل service boundary.
- **Structured logs:** pino JSON only، correlation با trace_id، PII redaction قبل ship.
- **Metrics:** Prometheus pull، custom business metrics (active_users, pages_created, ai_calls, mrr).
- **Synthetic monitoring:** Checkly/Uptime-Kuma من 3 مناطق جغرافية كل 60s.
- **Chaos:** scheduled chaos في staging أسبوعياً (kill pod, latency injection, DB failover).
- **Incident response:** PagerDuty/Opsgenie alternative مفتوح. Runbook لكل alert. blameless postmortem template.

### OpenTelemetry Bootstrap

```tsx
// /src/observability/otel.ts
import { NodeSDK } from '@opentelemetry/sdk-node'
import { OTLPTraceExporter } from '@opentelemetry/exporter-trace-otlp-http'
import { Resource } from '@opentelemetry/resources'
import { SemanticResourceAttributes } from '@opentelemetry/semantic-conventions'
import { getNodeAutoInstrumentations } from '@opentelemetry/auto-instrumentations-node'

export function startOtel(serviceName: string) {
	const sdk = new NodeSDK({
		resource: new Resource({
			[SemanticResourceAttributes.SERVICE_NAME]: serviceName,
			[SemanticResourceAttributes.SERVICE_VERSION]: process.env.GIT_SHA ?? 'dev',
			[SemanticResourceAttributes.DEPLOYMENT_ENVIRONMENT]: process.env.NODE_ENV ?? 'dev',
		}),
		traceExporter: new OTLPTraceExporter({
			url: process.env.OTEL_EXPORTER_OTLP_ENDPOINT,
			headers: { authorization: `Bearer ${process.env.OTEL_TOKEN}` },
		}),
		instrumentations: [getNodeAutoInstrumentations({
			'@opentelemetry/instrumentation-fs': { enabled: false },
		})],
	})
	sdk.start()
	process.on('SIGTERM', () => sdk.shutdown())
}
```

### Structured Logger (pino + correlation)

```tsx
// /src/observability/logger.ts
import pino from 'pino'
import { trace, context } from '@opentelemetry/api'

const PII_PATTERNS = [
	/[\w.-]+@[\w.-]+\.\w+/g,           // email
	/\b\d{3}-\d{2}-\d{4}\b/g,          // SSN
	/\b\d{13,19}\b/g,                  // credit card
	/\+?\d{10,15}/g,                   // phone
]

export const logger = pino({
	level: process.env.LOG_LEVEL ?? 'info',
	redact: ['*.password','*.token','*.authorization','*.cookie','req.body.password'],
	formatters: {
		log(obj) {
			const span = trace.getSpan(context.active())
			const ctx = span?.spanContext()
			return {
				...obj,
				trace_id: ctx?.traceId,
				span_id: ctx?.spanId,
			}
		},
	},
	hooks: {
		logMethod(args, method) {
			const sanitized = args.map(a => typeof a === 'string'
				? PII_PATTERNS.reduce((s, p) => s.replace(p, '[REDACTED]'), a)
				: a)
			return method.apply(this, sanitized as any)
		},
	},
})
```

### Burn-Rate Alerting (multi-window multi-burn-rate, Google SRE method)

```yaml
# /infra/prometheus/alerts.yml
groups:
- name: slo-burn-rate
  rules:
  # Fast burn: 2% budget in 1h (page immediately)
  - alert: APIAvailabilityFastBurn
    expr: |
      (
        slo_burn_rate{service="api",window="1h"} > 14.4
        and
        slo_burn_rate{service="api",window="5m"} > 14.4
      )
    for: 2m
    labels: { severity: page }
    annotations:
      summary: "API burning 2% of monthly error budget in 1h"
      runbook: "runbooks-url/api-availability"

  # Slow burn: 5% in 6h (ticket)
  - alert: APIAvailabilitySlowBurn
    expr: |
      slo_burn_rate{service="api",window="6h"} > 6
      and
      slo_burn_rate{service="api",window="30m"} > 6
    for: 15m
    labels: { severity: ticket }
```

### Runbooks (template per alert)

```markdown
# Runbook: API Availability Fast Burn

## Symptoms
- 5xx rate > 0.1% sustained
- Burn rate > 14.4× in 1h window

## Quick Diagnostics (5 min)
1. Check Grafana dashboard "API Health"
2. Recent deploys: `git log --since="1 hour ago"`
3. Check error breakdown: `logs-url?status=5xx`

## Immediate Actions
- [ ] Rollback last deploy if correlation exists
- [ ] Scale API replicas 2× if CPU > 70%
- [ ] Check DB connection pool saturation
- [ ] Verify upstream (Supabase/Mistral) status

## Escalation
If not resolved in 15 min → page secondary on-call
```
