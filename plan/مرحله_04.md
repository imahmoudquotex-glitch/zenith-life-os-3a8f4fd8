## B. Phase-Specific Deep Additions — Wave 03 (Security & Offline PWA)

### B.1 Vault Cryptography Upgrade — XChaCha20-Poly1305 بديلاً عن AES-GCM

- الـ baseline الجديد: **XChaCha20-Poly1305 (libsodium)** nonce 192-bit (24B). أوسع nonce space يسمح بـ random nonce reuse-safe.
- KDF: **Argon2id** (t=3, m=64MiB, p=4, salt=16B) حصرياً. PBKDF2 ممنوع.
- Master Key in WASM locked module (Sodium.js) مع `mlock`-equivalent memory protection.
- AEAD AAD context: `{workspace_id, item_id, version, key_version}`.
- **Key rotation:** key_version BIGINT في vault_items؛ روتين dual-write خلال migration period.
- Recovery: Shamir 3-of-5 (sss-wasm) opt-in. HW: WebAuthn `largeBlob`.

### B.2 CSP Hardening — Strict + Trusted Types

- أضف `require-trusted-types-for 'script'` + `trusted-types app default 'allow-duplicates'`.
- `script-src` يرفض `'unsafe-inline'` تماماً، ويعتمد على nonces و `'strict-dynamic'`.
- `script-src` يرفض `'unsafe-eval'` (Formula engine يستخدم interpreter خاص لا يتطلب eval — راجع W08).
- `connect-src` whitelist explicit: Supabase + Sentry (لو enabled) + Resend (لو enabled). أي domain جديد = ADR.
- CSP report-only أول أسبوعين في production ثم enforce.
- COOP `same-origin` + COEP `credentialless` + CORP `same-origin` لتمكين `SharedArrayBuffer` للـ WASM crypto.

### B.3 Service Worker Strict Cache Policy

- **NetworkOnly deny-list إجباري:** `/api/*`, `/auth/*`, `/vault/*`, `/account/*`, `/api/ai/*`, `/api/billing/*` (لو في)، `/api/webhooks/*`, `/api/csrf`, `/api/csp-report`, `/api/push/*`.
- **App shell precache:** `/`, `/offline`, manifest, fonts, icons (max 5MB).
- **Stale-while-revalidate:** فقط للـ static assets (images, public read pages).
- **Update flow:** skipWaiting + clientsClaim + UI prompt "تحديث متاح" مع manual approval.
- **Background sync** للـ outbox عند العودة للإنترنت.
- **ممنوع** cache لأي response فيها `Authorization` header أو Set-Cookie.

### B.5 CSRF Defense in Depth

- Origin/Referer check + double-submit cookie + SameSite=Lax + Sec-Fetch-Site.
- Sec-Fetch-Site يرفض أي request غير `same-origin`/`same-site` للـ mutations.
- CSRF token rotation عند login + كل 24h + عند password change.

### B.6 Audit Tamper-Evidence — Merkle Anchoring

- يومياً: `audit_daily_anchors(workspace_id, day, merkle_root BYTEA, event_count INT, created_at)`.
- Merkle root = SHA-256 لأحداث اليوم بترتيب (id, prev_hash, row_hash).
- اختياري: نشر الـ root على public blockchain (Bitcoin OP_RETURN أو Sigstore Rekor) للـ forensic-grade tamper evidence.

### B.7 Push Notifications — VAPID + Endpoint Hashing

- VAPID keys generated locally بـ `web-push generate-vapid-keys`.
- Endpoint stored بـ SHA-256 hash + الـ raw endpoint مشفر في DB (XChaCha20).
- Subscription auto-revoke عند `pushsubscriptionchange` event.
- رسائل push لا تحتوي على PII، فقط generic notification + URL موقّع.

- ❌ `localStorage.setItem` لأي token أو key (CI scan).
- ❌ Service Worker يحفظ response فيه `Set-Cookie`.
- ❌ CSP بدون `report-uri`/`report-to`.
- ❌ Web Crypto بدون isExtractable=false.
- ❌ VAPID private key في client bundle.
- ❌ Webhook handler بدون HMAC + nonce + timestamp + replay protection.

### HMAC Verification

```tsx
import { createHmac, timingSafeEqual } from 'node:crypto'

export function verifySignedPayload(args: {
  body: string
  signature: string
  timestamp: string
  secret: string
}) {
  const ageMs = Math.abs(Date.now() - Number(args.timestamp))
  if (ageMs > 5 * 60_000) throw new Error('SIGNATURE_EXPIRED')
  const expected = createHmac('sha256', args.secret)
    .update(`${args.timestamp}.${args.body}`)
    .digest('hex')
  const a = Buffer.from(args.signature.replace(/^sha256=/, ''), 'hex')
  const b = Buffer.from(expected, 'hex')
  if (a.length !== b.length || !timingSafeEqual(a, b)) throw new Error('BAD_SIGNATURE')
}
```

- ممنوع نهائيًا كتابة أي private key أو VAPID private أو CSP report token في Notion أو ملفات التوثيق أو رسائل التنفيذ.
- ممنوع حفظ أي secret in version control or project files. Secret scanning must always pass.
- Values of secrets should be in `.env.local` only, and this file must be gitignored.
- Non-secret values only are documented in `docs/preflight-w03.md` (e.g. VAPID public key, CSP allowlist, captcha site keys public).
- If any mandatory input is missing, Antigravity stops and writes: `BLOCKED: missing required input <name>`.
- If an optional input is missing, Antigravity writes: `STUB MODE: <feature> running without <provider>` in the migration header and in `docs/preflight-w03.md`.

### 5.1 Migration 0301 — Audit chain

```sql
-- 0301__audit_chain.sql
-- Wave: W03
BEGIN;
ALTER TABLE audit_events
  ADD COLUMN IF NOT EXISTS prev_hash BYTEA,
  ADD COLUMN IF NOT EXISTS row_hash  BYTEA;

CREATE OR REPLACE FUNCTION public.audit_compute_row_hash()
RETURNS TRIGGER LANGUAGE plpgsql AS $$
DECLARE v_prev BYTEA;
BEGIN
  SELECT row_hash INTO v_prev FROM audit_events
   WHERE workspace_id = NEW.workspace_id
   ORDER BY created_at DESC, id DESC LIMIT 1;
  NEW.prev_hash := COALESCE(v_prev, decode('00','hex'));
  NEW.row_hash := digest(
    coalesce(NEW.workspace_id,'') || '|' ||
    coalesce(NEW.actor_id,'') || '|' ||
    NEW.action || '|' ||
    coalesce(NEW.target_type,'') || '|' ||
    coalesce(NEW.target_id,'') || '|' ||
    NEW.created_at::text || '|' ||
    encode(NEW.prev_hash,'hex'),
    'sha256');
  RETURN NEW;
END $$;

DROP TRIGGER IF EXISTS trg_audit_chain ON audit_events;
CREATE TRIGGER trg_audit_chain BEFORE INSERT ON audit_events
FOR EACH ROW EXECUTE FUNCTION audit_compute_row_hash();

-- No UPDATE/DELETE on audit_events (confirm Wave 00 rules)
REVOKE UPDATE, DELETE ON audit_events FROM app_user;
COMMIT;
```

## 6. CSP & Security Headers Middleware

```tsx
// apps/web/src/middleware.ts
import 'server-only';
import { NextResponse, type NextRequest } from 'next/server';
import crypto from 'node:crypto';
import { serverEnv } from '@app/server-env';

function buildCsp(nonce: string): string {
  const supabaseHost = new URL(serverEnv.NEXT_PUBLIC_SUPABASE_URL).host;
  const directives = [
    `default-src 'self'`,
    `script-src 'self' 'nonce-${nonce}' 'strict-dynamic'`,
    `style-src 'self' 'nonce-${nonce}'`,
    `img-src 'self' data: blob: https:`,
    `font-src 'self' data:`,
    `connect-src 'self' https://${supabaseHost} wss://${supabaseHost}`,
    `worker-src 'self' blob:`,
    `manifest-src 'self'`,
    `frame-ancestors 'none'`,
    `base-uri 'self'`,
    `form-action 'self'`,
    `object-src 'none'`,
    `upgrade-insecure-requests`,
    `report-uri /api/csp-report`,
  ];
  return directives.join('; ');
}

export function middleware(req: NextRequest) {
  const nonce = crypto.randomBytes(16).toString('base64');
  const res = NextResponse.next();
  res.headers.set('Content-Security-Policy', buildCsp(nonce));
  res.headers.set('Strict-Transport-Security', 'max-age=63072000; includeSubDomains; preload');
  res.headers.set('X-Content-Type-Options', 'nosniff');
  res.headers.set('X-Frame-Options', 'DENY');
  res.headers.set('Referrer-Policy', 'strict-origin-when-cross-origin');
  res.headers.set('Permissions-Policy', 'camera=(), microphone=(self), geolocation=(), payment=(), usb=()');
  res.headers.set('Cross-Origin-Opener-Policy', 'same-origin');
  res.headers.set('Cross-Origin-Embedder-Policy', 'credentialless');
  res.headers.set('Cross-Origin-Resource-Policy', 'same-origin');
  res.headers.set('x-csp-nonce', nonce);
  res.headers.set('Server-Timing', `csp;desc="applied"`);
  return res;
}

export const config = {
  matcher: ['/((?!_next/static|_next/image|favicon.ico|sw.js|manifest.webmanifest).*)'],
};
```

## 7. HMAC Verification (No Timing Attack)

```tsx
// packages/security/src/hmac.ts
import 'server-only';
import crypto from 'node:crypto';

export function safeTimingEqual(a: string, b: string): boolean {
  const ab = Buffer.from(a, 'utf8');
  const bb = Buffer.from(b, 'utf8');
  if (ab.length !== bb.length) return false;     // Mandatory length guard
  return crypto.timingSafeEqual(ab, bb);
}

export type VerifyParams = {
  rawBody: string;
  timestamp: string;
  signature: string;
  secret: string;
  toleranceMs?: number;
};

export function verifySignedPayload(p: VerifyParams): { ok: boolean; reason?: string } {
  const tolerance = p.toleranceMs ?? 5 * 60 * 1000;
  const ts = Number(p.timestamp);
  if (!Number.isFinite(ts)) return { ok: false, reason: 'bad_timestamp' };
  const age = Math.abs(Date.now() - ts);
  if (age > tolerance) return { ok: false, reason: 'timestamp_skewed' };
  const expected = crypto.createHmac('sha256', p.secret)
    .update(`${p.timestamp}.${p.rawBody}`)
    .digest('hex');
  if (!safeTimingEqual(p.signature, expected)) return { ok: false, reason: 'signature_invalid' };
  return { ok: true };
}
```

```tsx
  if (!input) return fallback;
  if (!input.startsWith('/')) return fallback;
  if (input.startsWith('//')) return fallback;
  if (/[\r\n\\]/.test(input)) return fallback;
  try {
    const u = new URL(input, 'http://internal');
    if (u.hostname !== 'internal') return fallback;
  } catch { return fallback; }
  if (input.length > 512) return fallback;
  return input;
}
```

- Used in every auth callback (`/login`, `/signup`, `/oauth/*`).
- No full URLs in query params. Relative paths only.

## 12. Vault Client-Side E2E Encryption

### 12.1 Principle

- User enters passphrase. Never uploaded to server.
- Derive master key using **Argon2id** (params: memory=64MiB, iterations=3, parallelism=1, saltLen=16, keyLen=32).
- For each item: generate random 32-byte `item_key` → encrypt value with AES-GCM-256 (random 12B IV) → wrap `item_key` with master_key using AES-GCM to produce `wrapped_item_key`.
- Store in DB: `algo`, `iv`, `auth_tag`, `wrapped_item_key`, `ciphertext`, `aad` (workspace_id|user_id|item_id).
- Never store `master_key`. If user forgets passphrase → recovery code (Wave 03 does scaffolding, full activation in Wave 09).

### 12.2 Master key derivation

```tsx
// packages/vault-crypto/src/master-key.ts
import { argon2id } from './argon2';

export type KdfParams = { memory: number; iterations: number; parallelism: number; saltLen: number; keyLen: number };
export const DEFAULT_KDF: KdfParams = { memory: 65536, iterations: 3, parallelism: 1, saltLen: 16, keyLen: 32 };

export async function deriveMasterKey(passphrase: string, salt: Uint8Array, params = DEFAULT_KDF): Promise<Uint8Array> {
  if (passphrase.length < 12) throw new Error('passphrase_too_short');
  return argon2id({ password: passphrase, salt, ...params });
}
```

### 12.3 AES-GCM

```tsx
// packages/vault-crypto/src/aes-gcm.ts
export async function encryptAesGcm(key: Uint8Array, plaintext: Uint8Array, aad?: string): Promise<{ ciphertext: Uint8Array; iv: Uint8Array; }> {
  const iv = crypto.getRandomValues(new Uint8Array(12));
  const cryptoKey = await crypto.subtle.importKey('raw', key, 'AES-GCM', false, ['encrypt']);
  const ct = new Uint8Array(await crypto.subtle.encrypt({ name: 'AES-GCM', iv, additionalData: aad ? new TextEncoder().encode(aad) : undefined }, cryptoKey, plaintext));
  return { ciphertext: ct, iv };
}

export async function decryptAesGcm(key: Uint8Array, ciphertext: Uint8Array, iv: Uint8Array, aad?: string): Promise<Uint8Array> {
  const cryptoKey = await crypto.subtle.importKey('raw', key, 'AES-GCM', false, ['decrypt']);
  return new Uint8Array(await crypto.subtle.decrypt({ name: 'AES-GCM', iv, additionalData: aad ? new TextEncoder().encode(aad) : undefined }, cryptoKey, ciphertext));
}
```

### 12.4 Strict Rules

- No plaintext to server.
- No `master_key` or `item_key` in `localStorage`. Use session-only `IndexedDB` with a key wiped on `beforeunload`.
- AI must never read vault. `assertNoVaultContext` must be active before any AI call.
- No AES-CBC or RSA-PKCS1 v1.5 — AES-GCM only.

## 18. Outbox + Sync Engine

```tsx
// packages/offline/src/sync.ts
import { getOfflineDB, type OutboxRow } from './db';
import { calcNextAttempt } from './backoff';

const MAX_ATTEMPTS = 8;

export async function enqueueMutation(row: Omit<OutboxRow, 'status' | 'attempts' | 'createdAt' | 'nextAttemptAt'>) {
  const db = await getOfflineDB();
  await db.put('outbox', { ...row, status: 'pending', attempts: 0, createdAt: Date.now(), nextAttemptAt: Date.now() });
}

export async function processSyncQueue(fetchFn = fetch): Promise<{ ok: number; failed: number; dead: number }> {
  const db = await getOfflineDB();
  const stats = { ok: 0, failed: 0, dead: 0 };
  const tx = db.transaction('outbox', 'readwrite');
  const idx = tx.objectStore('outbox').index('byNextAttempt');
  const now = Date.now();
  let cursor = await idx.openCursor();
  while (cursor) {
    const row = cursor.value as OutboxRow;
    if (row.status === 'dead' || row.nextAttemptAt > now) { cursor = await cursor.continue(); continue; }
    await cursor.update({ ...row, status: 'syncing' });
    try {
      const res = await fetchFn(`/api/sync/${row.mutation}`, {
        method: 'POST',
        body: JSON.stringify(row.payload),
        headers: { 'Content-Type': 'application/json', 'Idempotency-Key': row.id, 'X-CSRF-Token': await readCsrfCookie() },
      });
      if (!res.ok) throw new Error(`HTTP ${res.status}`);
      await cursor.delete();
      stats.ok++;
    } catch (e) {
      const attempts = row.attempts + 1;
      const dead = attempts >= MAX_ATTEMPTS;
      await cursor.update({ ...row, status: dead ? 'dead' : 'failed', attempts, lastError: String(e), nextAttemptAt: calcNextAttempt(attempts) });
      dead ? stats.dead++ : stats.failed++;
    }
    cursor = await cursor.continue();
  }
  await tx.done;
  return stats;
}
```

```tsx
// packages/offline/src/backoff.ts
export function calcNextAttempt(attempts: number, now = Date.now()): number {
  // exponential with jitter ±20%, max 30 mins
  const base = Math.min(Math.pow(2, attempts) * 1000, 30 * 60 * 1000);
  const jitter = base * (0.8 + Math.random() * 0.4);
  return now + jitter;
}
```

## 19. Conflict Resolution

```tsx
// packages/offline/src/conflict.ts
export type Strategy = 'last_write_wins' | 'server_wins' | 'client_wins' | 'show_conflict';
export function getStrategy(entityType: string): Strategy {
  switch (entityType) {
    case 'task': return 'last_write_wins';
    case 'note': return 'show_conflict';
    case 'habit_checkin': return 'server_wins';
    case 'expense': return 'server_wins';
    case 'calendar_event': return 'last_write_wins';
    case 'vault_item': return 'server_wins';
    default: return 'last_write_wins';
  }
}
export function resolve(server: { version: number; updatedAt: string }, client: { version: number; updatedAt: string }, strategy: Strategy): 'client' | 'server' | 'show' {
  if (strategy === 'server_wins') return 'server';
  if (strategy === 'client_wins') return 'client';
  if (strategy === 'show_conflict') return server.version !== client.version ? 'show' : 'server';
  // last_write_wins
  if (client.version > server.version) return 'client';
  if (client.version < server.version) return 'server';
  return new Date(client.updatedAt) > new Date(server.updatedAt) ? 'client' : 'server';
}
```
