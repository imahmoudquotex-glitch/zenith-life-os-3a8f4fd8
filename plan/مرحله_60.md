### Failover Runbook State Machine

```tsx
export type FailoverState =
  | 'normal'
  | 'suspected_primary_failure'
  | 'read_only_mode'
  | 'queues_paused'
  | 'dns_switching'
  | 'secondary_promoted'
  | 'validation_running'
  | 'failed_over'
  | 'failback_planned'

export function nextFailoverState(state: FailoverState, signal: FailoverSignal): FailoverState {
  if (state === 'normal' && signal.primaryUnhealthyForMinutes >= 5) return 'suspected_primary_failure'
  if (state === 'suspected_primary_failure' && signal.onCallApproved) return 'read_only_mode'
  if (state === 'read_only_mode' && signal.writesDrained) return 'queues_paused'
  if (state === 'queues_paused' && signal.replicaLagSeconds < 30) return 'dns_switching'
  if (state === 'dns_switching' && signal.dnsPropagated) return 'secondary_promoted'
  if (state === 'secondary_promoted') return 'validation_running'
  if (state === 'validation_running' && signal.smokePassed) return 'failed_over'
  return state
}
```

### Safe Seed Rules

- ممنوع production dump raw في dev.
- synthetic tenants فقط مع `workspace_id` واضح.
- vault blobs fake encrypted ولا تحتوي أسرار حقيقية.
- seed يعيد نفس النتائج deterministic بـ seed ثابت.

## 🧠 Master Architect Upgrade — DR & Developer Parity Gate

- **Active-passive runbook:** وثّق trigger conditions، DNS switch، read-only mode، queue pause/resume، secret failover، وpost-failback validation.
- **Quarterly DR drills:** قياس RPO/RTO الفعلي كل ربع سنة، مع incident-style report وaction items.
- **Local parity:** docker-compose يشغل Postgres/RLS, Redis, MinIO/R2 mock, Mailhog, workers, search، وseed data tenant-safe خلال أقل من 5 دقائق.
- **Safe seeds:** لا production dump raw في dev. استخدم synthetic/anonymized fixtures، وvault blobs fake encrypted فقط.
- **Dev guardrails:** pre-commit يمنع direct AI provider calls, money float, missing RLS, console logs، وlight theme token drift.

## 🎯 نظرة عامة

المرحلة دي بتبني طبقتين أساسيتين: (1) Disaster Recovery كامل للوصول لـ RPO=5min و RTO=15min حتى لو رجعن إن region كامل بـ cross-region failover + active-passive setup + automated failover runbooks، (2) Local development environment دقيق يجعل أي developer جديد يقدر يشغل الـ stack كامل خلال 5 دقائق (docker-compose + seeded data + clear scripts). المرحلة دي بتقفل الدائرة: إحنا مستعدين لأسوأ السيناريوهات وفي نفس الوقت الفريق بيتحرك بسرعة في development.
