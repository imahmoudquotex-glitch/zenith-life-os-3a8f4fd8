## 🧠 تطوير تنفيذي إضافي — AI Gateway Production Pack

كل AI feature لازم يعدي من بوابة واحدة: quota، DLP، prompt isolation، tool approvals، وevals.

### Single Entrypoint

```tsx
export async function runAIWithQuota<T>(args: {
  workspaceId: string
  userId: string
  feature: 'summary'|'planner'|'goal_coach'|'search'|'agent'
  input: unknown
  schema: z.ZodSchema<T>
  tools?: ToolDefinition[]
  idempotencyKey: string
}): Promise<T> {
  const clean = await dlpSanitize(args.input)
  assertNoVaultOrE2EE(clean)
  const reservation = await aiQuota.reserve(args)
  try {
    const raw = await providerExecutor.complete({
      messages: buildIsolatedPrompt(clean),
      tools: filterAllowedTools(args.tools ?? [], args.feature),
    })
    const parsed = args.schema.parse(extractJson(raw))
    await aiQuota.complete(reservation, raw.usage)
    await aiAudit.write({ ...args, promptHash: hash(clean), responseHash: hash(raw) })
    return parsed
  } catch (e) {
    await aiQuota.rollback(reservation)
    throw e
  }
}
```

### Critical Rules

1.  **DLP (Data Loss Prevention):** أي نص قبل ما يروح للـ AI يعدي على `regex_scrubber` يشيل أرقام الكريدت كارد، الإيميلات، والأسماء الحقيقية لو مش مسموح بها.
2.  **Privacy Wall:** الـ `assertNoVaultOrE2EE` دي وظيفة بترمي Error فوري لو لقت أي data أصلها من Vault أو Workspace مشفر. الـ AI "أعمى" تماماً عن الـ Vault.
3.  **Mistral-First Pool:** هنستخدم Mistral API بـ 4 keys (Pool) بـ round-robin عشان نتفادى الـ Rate Limits في الـ free/pro tiers.
