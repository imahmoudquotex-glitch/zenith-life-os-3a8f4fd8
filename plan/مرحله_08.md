## B. Phase-Specific Deep Additions — Wave 07 (DB Engine & Relations)

### B.1 JSONB Properties Indexing Strategy

- GIN index على `properties JSONB` لـ wildcard search.
- **Generated columns** للـ properties الأكثر استخداماً (status, priority, due_date) بـ `GENERATED ALWAYS AS ((properties->>'status')::text) STORED` + btree index.
- Partial indexes على active rows (`WHERE is_deleted = false`).
- Expression indexes للـ date properties: `CREATE INDEX ON db_rows ((properties->>'due_date')::date)`.

### B.2 Property Type Codec Contract

```tsx

  type: PropertyType
  validateConfig(c: unknown): C       // throws on invalid
  encode(v: unknown, c: C): V | null  // strict input validation
  decode(raw: unknown, c: C): V | null
  toSearchText(v: V | null): string   // for tsvector
  toSortKey(v: V | null): string|number|null
  isVaultSensitive: boolean           // يمنع AI access
}
```

- Registry مغلقة لـ 19 type. أي type جديد = ADR.
- `isVaultSensitive: true` يرفع property للـ high sensitivity AI guard.

### B.3 Cross-Workspace Relation Lock-Down

- Database trigger `assert_relation_same_workspace()` (محدد في SQL fragment).
- Plus تحقق في service layer قبل INSERT لتجنب round-trip.
- pgTAP fuzz test: 1000 محاولة لـ cross-workspace relation → كلها تفشل.

### B.4 Rollup Server-Side Atomic

- Rollup compute عبر RPC فقط (لا client).
- Recompute trigger: عند relation change أو target row property change.
- Caching في Wave 08 `formula_cache` (توحيد الـ cache layer).
- pgTAP test لـ 6 aggregations (count/sum/avg/min/max/percent_checked).

### B.5 CSV Import Streaming Hardening

- `fast-csv` streaming مع backpressure handling.
- Max file 25MB، max rows 50,000 (soft warning) / 200,000 (hard limit).
- Virus scan stub (W19 integration).
- Header inference + manual column mapping.
- Validation per cell بـ property codec encode().
- Batch transactions 1000 rows + savepoints للـ partial success.
- SSE progress.
- Rejected rows report downloadable.
- **PII Detection:** بعد import، scan لـ CSV بحثاً عن SSN/credit cards/emails في properties غير معلنة كـ sensitive → warning.

### B.6 View Performance — 1000+ rows Desktop

- TanStack Table + react-virtual.
- Column virtualization على أكثر من 20 column.
- Sticky title column + sticky header.
- Keyboard nav: arrows بين cells (مثل Excel)، Enter للـ edit، Esc للـ cancel، Cmd+C/V للـ copy/paste.
- Bulk operations: select rows + bulk edit dialog + Plan Mode token (W20).
- Filter chip UI في Zenith style.

### B.7 Database Templates + AI Autofill Hook

- Template = page داخل الـ database (Wave 05 page system).
- Default template per DB.
- Hook لـ AI autofill (W20): `database_ai_fill_jobs(id, database_id, prompt, row_ids[], status, created_at)`.
- AI autofill تحترم vault sensitivity flags على properties.

### B.8 System Databases Versioning

- كل system DB (Tasks/Habits/Expenses/Notes/Goals/Projects) لها `system_db_schema_version INT`.
- Migrations توافق مع `is_system=true` constraint: ممنوع حذف properties أساسية أو تغيير أنواعها.
- Backfill script idempotent للـ existing workspaces.

### B.9 Anti-Patterns Additions

### Migration 0721 — db_inline_mode

```sql
ALTER TABLE databases
  ADD COLUMN IF NOT EXISTS layout_mode TEXT NOT NULL DEFAULT 'full_page'
    CHECK (layout_mode IN ('full_page','inline')),
  ADD COLUMN IF NOT EXISTS host_block_id TEXT REFERENCES blocks(id) ON DELETE SET NULL,
  ADD COLUMN IF NOT EXISTS host_page_id TEXT REFERENCES pages(id) ON DELETE SET NULL;

-- inline DB لازم يكون له host_block_id
ALTER TABLE databases ADD CONSTRAINT chk_inline_has_host
  CHECK (
    (layout_mode = 'inline' AND host_block_id IS NOT NULL AND host_page_id IS NOT NULL)
    OR (layout_mode = 'full_page' AND host_block_id IS NULL)
  );

CREATE INDEX idx_databases_host_page ON databases(host_page_id) WHERE layout_mode = 'inline';

-- RPC للتحويل بدون data loss
CREATE OR REPLACE FUNCTION convert_database_layout(p_db_id TEXT, p_target_mode TEXT, p_target_page_id TEXT DEFAULT NULL)
RETURNS VOID LANGUAGE plpgsql SECURITY DEFINER AS $$
DECLARE v_block_id TEXT;
BEGIN
  IF p_target_mode = 'inline' THEN
    IF p_target_page_id IS NULL THEN RAISE EXCEPTION 'inline_needs_host_page'; END IF;
    v_block_id := generate_ulid();
    INSERT INTO blocks(id, workspace_id, page_id, type, content_json, position, created_by_user_id, last_edited_by_user_id)
    VALUES (v_block_id, current_workspace_id(), p_target_page_id, 'database_inline',
            jsonb_build_object('database_id', p_db_id),
            (SELECT COALESCE(MAX(position),0)+1000 FROM blocks WHERE page_id = p_target_page_id),
            current_user_id(), current_user_id());
    UPDATE databases SET layout_mode='inline', host_block_id=v_block_id, host_page_id=p_target_page_id
      WHERE id=p_db_id AND workspace_id=current_workspace_id();
  ELSE
    UPDATE databases SET layout_mode='full_page', host_block_id=NULL, host_page_id=NULL
      WHERE id=p_db_id AND workspace_id=current_workspace_id();
  END IF;
END $$;
```

### Property Codec Interface

```tsx
export interface PropertyCodec<TValue, TConfig = unknown> {
  type: DbPropertyType
  validateConfig(config: unknown): TConfig
  encode(value: unknown, config: TConfig): TValue | null
  decode(raw: unknown, config: TConfig): TValue | null
  toSearchText(value: TValue | null): string
}

export const numberCodec: PropertyCodec<number, { format: 'number' | 'currency_cents' }> = {
  type: 'number',
  validateConfig: (c) => numberConfigSchema.parse(c),
  encode(value, config) {
    if (value == null) return null
    if (typeof value !== 'number' || !Number.isFinite(value)) throw new Error('INVALID_NUMBER')
    if (config.format === 'currency_cents' && !Number.isInteger(value)) throw new Error('MONEY_MUST_BE_CENTS')
    return value
  },
  decode: (raw) => (typeof raw === 'number' ? raw : null),
  toSearchText: (value) => value == null ? '' : String(value),
}
```

### Same-Workspace Relation Trigger

```sql
CREATE OR REPLACE FUNCTION assert_db_relation_same_workspace()
RETURNS TRIGGER LANGUAGE plpgsql AS $$
DECLARE src_ws TEXT; tgt_ws TEXT;
BEGIN
  SELECT workspace_id INTO src_ws FROM db_rows WHERE id = NEW.source_row_id;
  SELECT workspace_id INTO tgt_ws FROM db_rows WHERE id = NEW.target_row_id;
  IF src_ws IS NULL OR tgt_ws IS NULL OR src_ws <> tgt_ws THEN
    RAISE EXCEPTION 'RELATION_CROSS_WORKSPACE_BLOCKED';
  END IF;
  NEW.workspace_id := src_ws;
  RETURN NEW;
END $$;
```

```sql
  property_id TEXT NOT NULL REFERENCES db_properties(id) ON DELETE CASCADE,

  target_row_id TEXT NOT NULL REFERENCES db_rows(id) ON DELETE CASCADE,

CREATE POLICY db_relation_values_isolation ON db_relation_values USING (workspace_id = current_workspace_id());

-- Cross-workspace link prevention
CREATE OR REPLACE FUNCTION assert_relation_same_workspace() RETURNS TRIGGER LANGUAGE plpgsql AS $$
DECLARE v_src TEXT; v_tgt TEXT;
BEGIN
  SELECT workspace_id INTO v_src FROM db_rows WHERE id = NEW.source_row_id;
  SELECT workspace_id INTO v_tgt FROM db_rows WHERE id = NEW.target_row_id;
  IF v_src IS DISTINCT FROM v_tgt THEN RAISE EXCEPTION 'cross_workspace_relation_blocked'; END IF;
  RETURN NEW;
END $$;
CREATE TRIGGER trg_db_relation_same_ws BEFORE INSERT OR UPDATE ON db_relation_values
FOR EACH ROW EXECUTE FUNCTION assert_relation_same_workspace();

-- compute_rollup RPC
CREATE OR REPLACE FUNCTION compute_rollup(
  p_property_id TEXT,
  p_source_row_id TEXT,
  p_target_property_name TEXT,
  p_agg TEXT
) RETURNS JSONB LANGUAGE plpgsql STABLE SECURITY DEFINER
SET search_path = public, pg_temp AS $$
DECLARE v_result NUMERIC; v_count INT; v_text TEXT;
BEGIN
  IF p_agg = 'count' THEN
    SELECT COUNT(*) INTO v_count FROM db_relation_values rv
      WHERE rv.property_id = p_property_id AND rv.source_row_id = p_source_row_id;
    RETURN jsonb_build_object('value', v_count);
  ELSIF p_agg = 'percent_checked' THEN
    SELECT ROUND(100.0 * COUNT(*) FILTER (WHERE (r.properties->>p_target_property_name)::BOOLEAN)
                 / NULLIF(COUNT(*),0), 1) INTO v_result
      FROM db_relation_values rv JOIN db_rows r ON r.id = rv.target_row_id
      WHERE rv.property_id = p_property_id AND rv.source_row_id = p_source_row_id;
    RETURN jsonb_build_object('value', v_result);
  ELSIF p_agg IN ('sum','avg','min','max') THEN
    EXECUTE format('SELECT %s((r.properties->>%L)::NUMERIC) FROM db_relation_values rv JOIN db_rows r ON r.id = rv.target_row_id WHERE rv.property_id = %L AND rv.source_row_id = %L',
                   p_agg, p_target_property_name, p_property_id, p_source_row_id)
      INTO v_result;
    RETURN jsonb_build_object('value', v_result);
  ELSE
    RAISE EXCEPTION 'unsupported_aggregation';
  END IF;
END $$;
```

## 7. View Specs (مختصرة)

- **Table:** `@tanstack/react-table` + `@tanstack/react-virtual`. column resizing، sticky header، sticky title column، hidden columns، row height (compact/medium/tall).
- **Board:** dnd-kit columns by select/status property. drop = update + reorder. virtualized.
- **Gallery:** responsive grid 3/4/5 columns. cover field configurable. card properties subset.
- **Calendar:** date property as anchor. month/week/day. RTL-aware. drag-to-reschedule.
- **Timeline:** start+end date or duration. zoom levels (day/week/month/quarter).
- **List:** simplified table (no resize, no virtualization needed < 200 rows).
- **Gantt:** dependencies via relation property + critical path indicator.

## 8. Filter/Sort/Group Engine

- Filter operators per type: text(contains/equals/starts_with/ends_with/empty)، number(=/!=/>/</between)، date(is/before/after/this_week/last_30d/relative)، select(is/is_not/empty)، multi(contains_any/contains_all)، person(is_me/is/empty)، checkbox(checked/unchecked)، relation(contains/empty).
- AND/OR groups (nesting depth 3 max).
- Sort multi-key: properties array مع direction. ties broken by `created_at`.
- GroupBy: select/status/person/date(week/month).
- Server builds parametrized SQL (لا concatenation).

## 9. CSV Import

- Streaming `fast-csv` server-side. ممنوع `readFileSync` لـ > 1MB.
- Steps: upload → preview (10 rows) → column mapping → validate → import in batches 1000/transaction.
- Progress via SSE.
- Row error doesn't kill batch: rejected rows ترجع report.
- Max default 25MB. Max 50,000 rows soft warning.
