## B. Phase-Specific Deep Additions — Wave 08 (Formula Engine)

### B.1 Parser — LL(1) Recursive Descent (No Backtracking)

- LL(1) parser بدون backtracking → أسرع + أسهل في fuzz testing.
- Operator precedence: unary → power → mul/div → add/sub → comparison → and/or.
- Right-associative للـ power، left-associative للباقي.
- Max nesting depth 30 enforced في parser.
- Max identifier length 64 chars.
- Max number literal length 25 digits.
- Max string literal length 500 chars.

### B.2 Type System — Hindley-Milner Lite

- Type inference تلقائي للـ expressions.
- Generic types: `array<T>`.
- Type narrowing على `if` branches: `if(empty(x), 'default', concat(x, '!'))` → type narrow `x: text`.
- Strict equality: `==` يمنع type coercion (عكس JavaScript).
- Explicit casts: `toNumber()`, `toText()`, `toBoolean()`, `toDate()`.

### B.3 Function Registry — 50+ Functions (Expanded)

- **Text (10):** `concat`, `contains`, `lower`, `upper`, `length`, `slice`, `replace`, `trim`, `regexMatch` (whitelist patterns), `format`.
- **Numbers (15):** `add`, `sub`, `mul`, `div`, `round`, `floor`, `ceil`, `abs`, `min`, `max`, `pow`, `sqrt`, `mod`, `percentile`, `format`.
- **Dates (12):** `now`, `today`, `dateAdd`, `dateSub`, `dateBetween`, `formatDate`, `year`, `month`, `day`, `weekday`, `isoWeek`, `startOf`.
- **Logic (7):** `if`, `and`, `or`, `not`, `empty`, `equals`, `coalesce`.
- **Arrays (8):** `length`, `contains`, `join`, `first`, `last`, `unique`, `sort`, `filter`.
- **Rollups (8):** `count`, `sum`, `avg`, `min`, `max`, `percent_checked`, `percent_empty`, `median`.

### B.4 Deterministic Execution

- `now()` يأخذ snapshot من `context.now` — ثابت خلال evaluation واحدة.
- `today()` = `now().date()` في `context.userTimezone`.
- `random()` ممنوع (non-deterministic) — لو لازم، استخدم seeded variant: `seededRandom(seed)`.
- Float operations تستخدم IEEE 754 standard، مع `round()` explicit للـ money operations.

### B.5 Evaluation Sandbox — Web Worker (Client) + Node VM-Lite (Server)

- **Client preview:** Web Worker isolated، timeout 100ms، terminate on exceeded.
- **Server authoritative:** Node بدون vm module (تجنب escape attacks)، بدلاً: manual AST walker مع timeout via `AbortSignal` + iteration counter.
- Iteration counter: max 10,000 عملية per evaluation.
- Memory budget: max 5MB intermediate values.

### B.6 Vault + Cross-Workspace Guards

- أي property reference لـ vault-sensitive property → throw `FORMULA_VAULT_ACCESS_DENIED`.
- أي relation لـ row في workspace ثاني → throw `FORMULA_CROSS_WORKSPACE_DENIED`.
- Static analysis في parser: لو AST يرجع لـ property في database ثاني، تحقق من same workspace.

### B.7 Cache + Recalc Strategy

- Cache key: `(row_id, property_id)`.
- Invalidation عند:
    - Row property update (أي property في dependencies).
    - Schema change (formula definition update).
    - Related row property update (عبر relation).
    - Time-based (لو الـ formula بتستخدم `now()`/`today()`): TTL = end of day في userTimezone.
- Recalc queue: BullMQ مع priority (interactive=high, schema-change=critical, time-based=low).
- Stale indicator في UI + last_computed_at.

### B.8 Audit Diff

- كل formula change يسجّل `audit_events` مع `before/after` expressions + AST diff.
- Approval workflow optional لـ formulas تتأثر على money columns.

### B.9 Anti-Patterns Additions

- ❌ أي `eval`, `Function`, `new Function`, `setTimeout` داخل engine.
- ❌ Dynamic property access via string (`row[prop_name]` غير type-checked).
- ❌ HTTP fetch داخل formula (رفض في parser).
- ❌ File system access (impossible في Web Worker، لكن أي require/import في server eval = blocker).
- ❌ Recursive function calls غير محدودة.
- ❌ Non-deterministic functions (Math.random)، ما عدا seeded variants.

### Tokenizer Guard

```tsx
const BLOCKED_TOKENS = ['eval', 'Function', 'import', 'require', '__proto__', 'constructor', 'prototype']

export function tokenize(input: string): Token[] {
  if (input.length > 2000) throw new FormulaError('FORMULA_LENGTH_EXCEEDED')
  for (const bad of BLOCKED_TOKENS) {
    if (input.includes(bad)) throw new FormulaError('FORMULA_UNSAFE_TOKEN')
  }
  return scanTokens(input)
}
```

```tsx
export type FormulaFn = {
  name: string
  args: FormulaType[]
  returns: FormulaType
  deterministic: true

}

export const registry = new Map<string, FormulaFn>([
  ['if', fnIf],
  ['concat', fnConcat],

```

```tsx
export async function evaluateWithBudget(ast: FormulaNode, ctx: EvalContext) {
  const started = performance.now()
  const result = await evaluateNode(ast, ctx)
  if (performance.now() - started > 50) throw new FormulaError('FORMULA_TIMEOUT')
  return result
}
```

## 6. AST + Parser

```tsx
export type FormulaNode =
  | { type: 'literal'; valueType: 'number' | 'text' | 'boolean' | 'null'; value: unknown }
  | { type: 'property'; propertyId: string }
  | { type: 'call'; name: string; args: FormulaNode[] }
  | { type: 'binary'; op: '+'|'-'|'*'|'/'|'%'|'='|'!='|'>'|'<'|'>='|'<='; left: FormulaNode; right: FormulaNode }
  | { type: 'unary'; op: '-' | '!'; arg: FormulaNode };

export function parseFormula(expression: string): FormulaNode {
  assertLength(expression, 2000);
  const tokens = tokenize(expression);            // disallow dangerous chars
  const ast = parseExpression(tokens);
  assertDepth(ast, 30);
  assertAllowedFunctions(ast);                    // whitelist from registry
  return ast;
}
```

قواعد parser: ممنوع `eval`، ممنوع `Function`، ممنوع `__proto__`، ممنوع `Object.prototype`، ممنوع identifiers أطول من 64 char.

## 7. Type Checker

- Static type inference per node.
- Function signatures مع overloads (e.g. `add(number, number)→number`، `concat(text, text)→text`).
- Errors بـ codes: `FORMULA_TYPE_MISMATCH`, `FORMULA_UNKNOWN_FUNCTION`, `FORMULA_ARITY_WRONG`, `FORMULA_PROPERTY_NOT_FOUND`, `FORMULA_CYCLE_DETECTED`, `FORMULA_DEPTH_EXCEEDED`, `FORMULA_LENGTH_EXCEEDED`, `FORMULA_TIMEOUT`, `FORMULA_VAULT_ACCESS_DENIED`.

## 8. Evaluator

```tsx
export async function evaluateFormula(opts: {
  ast: FormulaNode;
  row: Record<string, unknown>;
  relations: Record<string, Array<Record<string, unknown>>>;
  context: { now: Date; userTimezone: string; userId: string };
  timeoutMs?: number;
}): Promise<{ value: unknown; resultType: string }> {
  return withTimeout(evaluateNode(opts.ast, opts), opts.timeoutMs ?? 50);
}
```

قواعد:

- ممنوع `eval`/`Function`/`new Function`.
- ممنوع side effects.
- vault property access → throw `FORMULA_VAULT_ACCESS_DENIED`.
- `now()`/`today()` يستخدمون `context.now` (snapshot، deterministic).

## 9. Dependency Graph + Cycle Detection

- `dep-extractor.ts`: AST → `Set<propertyId>`.
- على save:
    1. Parse + type-check.
    2. Extract deps.
    3. Build graph من كل formula_definitions في الـ DB.
    4. Topo sort. لو cycle → reject.
    5. حفظ definition + invalidate cache للـ dependents.
    6. enqueue recalc.
- على row update لـ property X: invalidate cache لكل formulas اللي بتعتمد على X في نفس الصف.

## 10. Recalc Queue

- BullMQ queue `formula-recalc` (لو Redis ناقص → DB-based polling worker).
- Job payload: `{ workspaceId, databaseId, propertyId, rowIds[]? }`.
- Worker concurrency 4 default.
- Batch 1000 rows per job. Failure → retry بـ exponential backoff.
- Metrics في `recalc_metrics` (duration p50/p95/p99، failures).
- UI shows stale indicator + last computed_at.
