## B. Phase-Specific Deep Additions — Wave 10 (Habits + Finance)

### B.1 Money Primitives — Cents + BPS Strict

- كل money column = `*_cents BIGINT NOT NULL CHECK (* >= 0)`.
- FX rates = `*_bps INT NOT NULL CHECK (* > 0)` (10000 = 1.0).
- Parser في `packages/money/src/cents.ts`: `parseUserInput()` يدعم `1,500.50`, `1.5k`, `٢٠٠٠`, `EGP 200`, scientific notation rejected.
- Currency: `CHAR(3)` ISO 4217 + CHECK constraint (USD/EUR/GBP/EGP/SAR/AED/...).
- CI scan `check-money-columns.ts` يفحص migrations + repos.
- `formatCents(cents, currency, locale)` يستخدم `Intl.NumberFormat` فقط (لا manual formatting).

### B.2 Habit Streak Timezone Correctness

- `local_day(user_id)` PostgreSQL function تستخدم `user.timezone` (IANA).
- Future checkin blocked في RPC `toggle_habit_checkin` (لا client guard فقط).
- DST handling: `day_local` يتحسب في user timezone، لا UTC.
- UNIQUE (user_id, habit_id, day_local) لمنع duplicate checkins.
- Streak calculator stateful (memoization per user-month) لتجنب N+1.

### B.3 Atomic Budget Check (No Race Conditions)

- RPC `add_expense_with_budget_check` في transaction واحدة:
    1. Lock budget row `FOR UPDATE`.
    2. Idempotency lookup.
    3. Compute current spent.
    4. Insert expense.
    5. Return remaining/exceeded status.
- Policy: exceeded warning (لا block) — الـ user يقرر التجاوز.
- Audit per expense + budget change.

### B.4 AI Finance Features — Strict Quota

- كل AI feature خلف `runAIWithQuota({ kind: 'finance_*' })`:
    - `finance_quick_parse` (500/month/user)
    - `finance_voice_transcribe` (100/month/user)
    - `finance_receipt_extract` (50/month/user)
    - `finance_auto_categorize` (1000/month/user)
    - `finance_monthly_review` (12/year/user)
- Cache aggressively: `finance_quick_parse_cache` (TTL 24h), `finance_ai_insights_cache` (TTL per kind).
- Whisper API privacy: audio deleted after transcription, no storage.
- Receipt OCR: Tesseract.js client-first, LLM fallback if confidence < 70%.
- Sensitive data (account numbers, full card numbers) NEVER sent to AI.

### B.5 Voice Privacy & Permissions

- Mic permission requested explicitly first time.
- Audio recording max 30s.
- Audio buffer cleared after transcription.
- No audio file stored on server (server-side: stream to Whisper, discard).
- A11y: `aria-pressed`, `aria-live` for transcript preview.

### B.6 FX Rates Daily Snapshot Safety

- Daily worker fetches rates from provider (with fallback to yesterday).
- BPS storage immutable per (date, base, target).
- Audit source + timestamp.
- Expense uses rate snapshot at create time (locked in, no retroactive change).
- Fallback: stub rates for offline dev (documented).

### B.7 Habits Primitives Only (UI in W10.5)

- W10 builds: `habit_checkins`, `toggle_habit_checkin` RPC, `habit_streaks_view`, archive guard.
- W10 does NOT build: HabitGrid, HabitHeatmap, MonthlyHabitsTracker — those are W10.5.
- ESLint rule: `packages/habits-ui` import rejected in W10 commits.

### B.8 Anti-Patterns Additions

- ❌ FLOAT/NUMERIC/DECIMAL في money path.
- ❌ `parseFloat`/`Number()` على amount input.
- ❌ Streak في UTC.
- ❌ Future checkin.
- ❌ Budget update غير atomic.
- ❌ Hard delete expense.
- ❌ Expense بدون Idempotency-Key.
- ❌ CSV غير streaming.
- ❌ Habits UI in W10 commits.
- ❌ AI Finance feature بدون runAIWithQuota.
- ❌ Audio storage server-side.
- ❌ Sensitive PII sent to AI without redaction.

### Money Parser Without Float

```tsx
export function parseAmountToCents(raw: string): bigint {
  const normalized = raw.trim().replace(/[,_\s]/g, '')
  const match = normalized.match(/^(\d+)(?:\.(\d{1,2}))?$/)
  if (!match) throw new Error('INVALID_MONEY_AMOUNT')
  const pounds = BigInt(match[1])
  const cents = BigInt((match[2] ?? '').padEnd(2, '0'))
  return pounds * 100n + cents
}
```

### Atomic Expense RPC

```sql
CREATE OR REPLACE FUNCTION add_expense_with_budget_check(
  p_workspace_id TEXT,
  p_user_id TEXT,
  p_idempotency_key TEXT,
  p_amount_cents BIGINT,
  p_category TEXT
) RETURNS TEXT LANGUAGE plpgsql SECURITY DEFINER SET search_path = public, pg_temp AS $$
DECLARE v_id TEXT := generate_ulid();
BEGIN
  PERFORM 1 FROM idempotency_keys WHERE workspace_id = p_workspace_id AND key = p_idempotency_key FOR UPDATE;
  IF FOUND THEN RETURN (SELECT response_json->>'expense_id' FROM idempotency_keys WHERE key = p_idempotency_key); END IF;
  INSERT INTO expenses(id, workspace_id, user_id, amount_cents, category)
  VALUES (v_id, p_workspace_id, p_user_id, p_amount_cents, p_category);
  PERFORM check_budget_warning(p_workspace_id, p_user_id, p_category);
  RETURN v_id;
END $$;
```

### Habit Toggle Guard

```sql
IF p_day_local > (now() AT TIME ZONE v_user_timezone)::date THEN
```

### 4.5.3 Data Source

    ```sql
    WHERE user_id = $1
      AND day_local >= $2  -- first of month
      AND day_local <= $3  -- last of month
    ```

### 4.6.2 QuickAddHero — مواصفات الـ Parser الذكي

- Input واحد بـ placeholder rotating: `"2000 أكل"` / `"اشتريت قهوة بـ 50"` / `"1.5k مواصلات أمس"`.
- Global keyboard shortcut: `Q` من أي مكان في `/finance` → focus على Hero input.
- **Grammar يدعم:**
    - **Number:** `2000` / `1.5k` / `1,500` / `1500.50` / `2K` / `٢٠٠٠` (Arabic numerals).
    - **Currency:** `جنيه` / `EGP` / `$` / `دولار` / `USD` / `€` / `يورو` (default = workspace).
    - **Category hints:** أي كلمة عربية → AI auto-matches (أكل، قهوة، مواصلات، إيجار، فاتورة، نت، بنزين، تسوق، صحة، ترفيه، هدية، تبرع، ...).
    - **Time hints:** `أمس` / `أول أمس` / `الإثنين الماضي` / `قبل 3 أيام` / `02/05` (default = today).
    - **Merchant hints:** `من كارفور` / `في ماكدونالدز` → merchant=X.
    - **Payment:** `كاش` / `فيزا` / `إنستاباي` / `محفظة` → payment_method=X.
- **Examples runtime-tested:**
    - `"2000 أكل"` → `{ amount_cents: 200000, category: "أكل", date: today, payment: last_used, currency: EGP }`
    - `"50 قهوة كاش من كافيه أمس"` → `{ amount_cents: 5000, category: "قهوة", date: yesterday, merchant: "كافيه", payment: "كاش" }`
    - `"1.5k إيجار شهر مايو"` → `{ amount_cents: 150000, category: "إيجار", description: "شهر مايو", recurring_candidate: true }`
    - `"٢٠٠ بنزين شل فيزا"` → `{ amount_cents: 20000, category: "بنزين", merchant: "شل", payment: "فيزا" }`
- **Fallback:** لو AI parser failed (نص غير معروف) → side drawer (مش modal) بـ 3 حقول فقط: amount / category / date — الباقي defaults.
- **Latency target:** parse + insert < 350ms p95 (cache lookups).
- **Powered by:** `runAIWithQuota({ userId, kind: 'finance_quick_parse', sub: text })` — quota 500 parse/شهر/user.

### 5.2 Migration 1002 — toggle_habit_checkin RPC

```sql
CREATE OR REPLACE FUNCTION toggle_habit_checkin(
  p_habit_id TEXT,
  p_day DATE DEFAULT NULL
) RETURNS TABLE(checked BOOLEAN, day_local DATE) LANGUAGE plpgsql SECURITY DEFINER
SET search_path = public, pg_temp AS $$
DECLARE
  v_user TEXT := current_user_id();
  v_today DATE;
  v_day DATE;
  v_existed BOOLEAN;
BEGIN
  v_today := local_day(v_user);
  v_day := COALESCE(p_day, v_today);
  IF v_day > v_today THEN RAISE EXCEPTION 'future_checkin_not_allowed'; END IF;

  DELETE FROM habit_checkins WHERE user_id=v_user AND habit_id=p_habit_id AND day_local=v_day
    RETURNING TRUE INTO v_existed;
  IF v_existed THEN
    RETURN QUERY SELECT FALSE, v_day; RETURN;
  END IF;

  INSERT INTO habit_checkins(id, workspace_id, user_id, habit_id, day_local)
  VALUES (ulid(), current_workspace_id(), v_user, p_habit_id, v_day)
  ON CONFLICT (user_id, habit_id, day_local) DO NOTHING;
  RETURN QUERY SELECT TRUE, v_day;
END $$;
```

```sql
CREATE OR REPLACE FUNCTION expenses_validate_money() RETURNS TRIGGER LANGUAGE plpgsql AS $$
DECLARE v_is_expenses BOOLEAN; v_amount NUMERIC; v_bps INT;
BEGIN
  SELECT (title = 'Expenses' AND is_system) INTO v_is_expenses FROM databases WHERE id = NEW.database_id;
  IF NOT v_is_expenses THEN RETURN NEW; END IF;
  v_amount := (NEW.properties->>'amount_cents')::NUMERIC;

    RAISE EXCEPTION 'invalid_fx_rate_bps'; END IF;

END $$;
```

### 5.4 Migration 1007 — add_expense_with_budget_check RPC

```sql
CREATE OR REPLACE FUNCTION add_expense_with_budget_check(
  p_budget_row_id TEXT,
  p_amount_cents BIGINT,
  p_currency TEXT,
  p_category TEXT,
  p_merchant TEXT,
  p_description TEXT,
  p_date DATE,
  p_idempotency_key TEXT
) RETURNS JSONB LANGUAGE plpgsql SECURITY DEFINER
SET search_path = public, pg_temp AS $$
DECLARE
  v_user TEXT := current_user_id();
  v_ws TEXT := current_workspace_id();
  v_budget_amount BIGINT;
  v_spent BIGINT;
  v_expense_id TEXT;
BEGIN
  IF p_amount_cents <= 0 THEN RAISE EXCEPTION 'amount_must_be_positive'; END IF;

  -- Idempotency lookup في db_rows تبع Expenses
  SELECT id INTO v_expense_id FROM db_rows
   WHERE workspace_id=v_ws AND properties->>'idempotency_key'=p_idempotency_key AND NOT is_deleted;
  IF FOUND THEN
    RETURN jsonb_build_object('expense_row_id', v_expense_id, 'replayed', TRUE);
  END IF;

  -- Budget lock
  IF p_budget_row_id IS NOT NULL THEN
    SELECT (properties->>'amount_cents')::BIGINT INTO v_budget_amount FROM db_rows
     WHERE id = p_budget_row_id AND workspace_id=v_ws FOR UPDATE;
    SELECT COALESCE(SUM((properties->>'amount_cents')::BIGINT), 0) INTO v_spent FROM db_rows
     WHERE workspace_id=v_ws AND properties->>'budget_row_id'=p_budget_row_id AND NOT is_deleted;
  END IF;

  -- Insert expense row
  v_expense_id := ulid();
  INSERT INTO db_rows(id, workspace_id, database_id, properties, created_by_user_id, last_edited_by_user_id, position)
  VALUES (v_expense_id, v_ws, (SELECT id FROM databases WHERE workspace_id=v_ws AND title='Expenses' AND is_system),
    jsonb_build_object(
      'amount_cents', p_amount_cents,
      'currency', p_currency,
      'fx_rate_bps', 10000,
      'category', p_category,
      'merchant', p_merchant,
      'description', p_description,
      'expense_date', p_date,
      'budget_row_id', p_budget_row_id,
      'idempotency_key', p_idempotency_key
    ),
    v_user, v_user, extract(epoch from now()));

  v_spent := COALESCE(v_spent, 0) + p_amount_cents;

  RETURN jsonb_build_object(
    'expense_row_id', v_expense_id,
    'budget_used_cents', v_spent,
    'budget_remaining_cents', GREATEST(COALESCE(v_budget_amount,0) - v_spent, 0),
    'exceeded', COALESCE(v_budget_amount,0) > 0 AND v_spent > v_budget_amount
  );
END $$;
```

## 6. Streak Calculator (timezone-aware)

```tsx
// packages/habits/src/streak.ts
export function calculateStreak(
  checkinDays: string[],     // sorted desc ISO YYYY-MM-DD in user TZ
  todayInTz: string
): { current: number; longest: number } {
  if (!checkinDays.length) return { current: 0, longest: 0 };
  let current = 0, longest = 0, streak = 0;
  let prev: string | null = null;
  for (const d of checkinDays) {
    if (prev === null) {
      streak = (d === todayInTz || dayDiff(todayInTz, d) === 1) ? 1 : 0;
    } else {
      const diff = dayDiff(prev, d);
      streak = diff === 1 ? streak + 1 : 1;
    }
    prev = d;
    longest = Math.max(longest, streak);
  }
  const first = checkinDays[0];
  current = (first === todayInTz || dayDiff(todayInTz, first) === 1) ? longest : 0;
  return { current, longest };
}
function dayDiff(newer: string, older: string): number {
  return Math.round((Date.parse(newer) - Date.parse(older)) / 86_400_000);
}
```

## 7. Money Helpers (Cents + BPS)

```tsx
// packages/money/src/cents.ts
export function parseUserInput(input: string): number {
  const clean = input.replace(/[^\d.]/g, '');
  const parts = clean.split('.');
  const whole = parseInt(parts[0] || '0', 10);
  const frac = parseInt((parts[1] ?? '').padEnd(2, '0').slice(0, 2), 10);
  if (Number.isNaN(whole) || Number.isNaN(frac)) throw new Error('invalid_amount');
  return whole * 100 + frac;                       // BIGINT-safe under JS limits
}
export function formatCents(cents: number, currency = 'EGP', locale = 'ar-EG'): string {
  return new Intl.NumberFormat(locale, { style: 'currency', currency, minimumFractionDigits: 2 })
    .format(cents / 100);
}
export function addCents(...amounts: number[]): number { return amounts.reduce((a, b) => a + b, 0); }

// packages/money/src/basis-points.ts
export function ratioToBps(ratio: number): number { return Math.round(ratio * 10_000); }
export function bpsToRatio(bps: number): number { return bps / 10_000; }
export function convertCents(cents: number, rateBps: number): number {
  return Math.round((cents * rateBps) / 10_000);
}
```

لا `parseFloat` على amounts. لا NaN propagation داخل `formatCents`.

### Widget Data Contract

```tsx
// src/features/finance/lib/get-money-flow.ts
import 'server-only'
import { unstable_cache } from 'next/cache'

interface MoneyFlowData {
  todaySpentCents:        number
  todayBudgetCents:       number
  weekSpentCents:         number
  monthSpentCents:        number
  monthBudgetCents:       number
  topCategory:            { name: string; cents: number; pct: number }
  recentExpenses:         Array<{ id: string; amountCents: number; category: string; merchant?: string; createdAt: string }>
  sparkline7d:            number[]  // last 7 days totals, oldest first
}

export const getMoneyFlow = unstable_cache(
  async (userId: string): Promise<MoneyFlowData> => {
    const today = new Date().toISOString().slice(0, 10)
    const weekStart = new Date(Date.now() - 7 * 86_400_000).toISOString().slice(0, 10)
    const monthStart = today.slice(0, 7) + '-01'

    const [todayExp, weekExp, monthExp, budgets, recent, byCat, spark] = await Promise.all([
      fetchExpenseSum(userId, today, today),
      fetchExpenseSum(userId, weekStart, today),
      fetchExpenseSum(userId, monthStart, today),
      fetchActiveBudgets(userId, monthStart),
      fetchRecentExpenses(userId, 5),
      fetchByCategoryThisMonth(userId, monthStart),
      fetch7DayDailyTotals(userId, weekStart),
    ])

    const top = byCat[0]
    return {
      todaySpentCents: todayExp.total,
      todayBudgetCents: budgets.dailyTotal,
      weekSpentCents: weekExp.total,
      monthSpentCents: monthExp.total,
      monthBudgetCents: budgets.monthlyTotal,
      topCategory: top
        ? { name: top.category, cents: top.cents, pct: Math.round((top.cents / monthExp.total) * 100) }
        : { name: '—', cents: 0, pct: 0 },
      recentExpenses: recent,
      sparkline7d: spark,
    }
  },
  ['money-flow'],
  { revalidate: 60, tags: ['finance', 'money-flow'] }
)
```

### 5.2 Migration 1107 — generate_monthly_archive RPC

```sql
CREATE OR REPLACE FUNCTION generate_monthly_archive(
  p_year_month TEXT DEFAULT NULL
) RETURNS JSONB LANGUAGE plpgsql SECURITY DEFINER
SET search_path = public, pg_temp AS $$
DECLARE
  v_user TEXT := current_user_id();
  v_ws TEXT := current_workspace_id();
  v_ym TEXT;
  v_start DATE; v_end DATE;
  v_snapshot JSONB;
  v_id TEXT;
BEGIN
  v_ym := COALESCE(p_year_month, to_char(local_day(v_user) - INTERVAL '1 day', 'YYYY-MM'));
  v_start := to_date(v_ym || '-01', 'YYYY-MM-DD');
  v_end := (v_start + INTERVAL '1 month - 1 day')::DATE;

  WITH base AS (
    SELECT habit_id, COUNT(*) AS checkins
    FROM habit_checkins
    WHERE user_id = v_user AND day_local BETWEEN v_start AND v_end
    GROUP BY habit_id
  )
  SELECT jsonb_build_object(
    'year_month', v_ym,
    'total_checkins', COALESCE(SUM(checkins), 0),
    'days_in_month', (v_end - v_start + 1),
    'by_habit', COALESCE(jsonb_agg(jsonb_build_object('habit_id', habit_id, 'checkins', checkins) ORDER BY checkins DESC), '[]'::jsonb),
    'completion_pct', NULL,    -- computed in service layer
    'top_habit', (SELECT habit_id FROM base ORDER BY checkins DESC LIMIT 1),
    'weakest_habit', (SELECT habit_id FROM base ORDER BY checkins ASC LIMIT 1)
  ) INTO v_snapshot FROM base;

  v_id := ulid();
  INSERT INTO habit_monthly_archives(id, workspace_id, user_id, year_month, snapshot)
  VALUES (v_id, v_ws, v_user, v_ym, v_snapshot)
  ON CONFLICT (user_id, year_month) DO UPDATE SET snapshot = EXCLUDED.snapshot, generated_at = now()
  RETURNING id INTO v_id;

  RETURN jsonb_build_object('archive_id', v_id, 'year_month', v_ym, 'snapshot', v_snapshot);
END $$;
```

```sql
CREATE OR REPLACE VIEW habit_long_streaks_view AS
WITH ranked AS (
  SELECT
    habit_id, user_id, day_local,
    day_local - (ROW_NUMBER() OVER (PARTITION BY habit_id, user_id ORDER BY day_local))::INT AS streak_group
  FROM habit_checkins
)
SELECT
  user_id, habit_id,
  MAX(day_local) AS last_day,
  COUNT(*) AS streak_length,
  MIN(day_local) AS streak_start
FROM ranked
GROUP BY user_id, habit_id, streak_group
HAVING COUNT(*) >= 7      -- 🏆 min 7-day cutoff
  AND MAX(day_local) >= (CURRENT_DATE - INTERVAL '1 day')  -- active only
ORDER BY streak_length DESC;
```

## 11. AI Coach Architecture

كل AI feature بنفس النمط:

```tsx
async function runHabitsCoach(userId: string, sub: HabitsCoachSub, payload: unknown) {
  // 1. Check cache
  const cached = await getInsight(userId, sub, payload);
  if (cached && !cached.expired) return cached.content;
  // 2. Quota check
  const allowed = await runAIWithQuota({ userId, kind: 'habits_coach', sub });
  if (!allowed) throw new Error('quota_exceeded');
  // 3. Build prompt (deterministic + scoped)
  const prompt = buildPrompt(sub, payload, userContext);
  // 4. Call AI provider (single retry, 10s timeout)
  const result = await aiProvider.complete(prompt);
  // 5. Cache + audit + return
  await saveInsight(userId, sub, payload, result, ttlFor(sub));
  return result;
}
```

قواعد:

- ممنوع AI call في render (server-only).
- كل AI call له audit row.
- TTL حسب الـ sub:
    - daily_nudge: 24h
    - weekly_summary: 7d
    - monthly_review: 30d
    - pattern_detection: 14d
    - timing_optimization: 14d
    - streak_recovery: 24h (until recovered)
    - stacking/template_suggestion: 30d
    - difficulty_adjust: 14d
    - motivation/day_of_week_insights/month_comparison/goal_alignment/long_term_trends/habit_bundling: 7d
    - health_score: 24h
- Refresh manual: زر "تحديث" في كل widget → bust cache + ري-run.
