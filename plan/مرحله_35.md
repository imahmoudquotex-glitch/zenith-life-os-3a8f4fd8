### Claim Verification Script

```tsx
const CLAIMS = [
  { phrase: 'صفر-معرفة', requires: ['docs/security/vault.md', 'packages/vault-crypto'] },
  { phrase: 'مجاني للأبد', requires: ['docs/product/free-forever.md'] },
  { phrase: 'AI آمن', requires: ['packages/ai-gateway', 'docs/security/ai-dlp.md'] },
]

export function assertMarketingClaims(copy: string) {
  for (const claim of CLAIMS) {
    if (copy.includes(claim.phrase)) {
      for (const req of claim.requires) assertPathExists(req)
    }
  }
}
```

## 🧠 Master Architect Upgrade — Legal/Marketing Integrity Gate

- **Claims must match product:** أي ادعاء "صفر-معرفة/E2EE/AI/private/free forever" لازم يكون مدعوم بتنفيذ فعلي ورابط security docs، وإلا يتحول لصياغة أدق.
- **No dark marketing:** ممنوع urgency/scarcity/social comparison/leaderboards. testimonials اختيارية ومجهولة/موافق عليها.
- **Compliance pages:** Privacy/Terms/DPA/Cookies/AUP لازم تغطي GDPR/CCPA، data deletion/export، AI data use، integrations processors، breach notification، وsubprocessors.
- **Security page:** وضّح RLS, E2EE vault, backups, audit logs, vulnerability disclosure، وحدود ما هو غير مشفر end-to-end.
- **Public performance:** marketing routes SSG/ISR، no heavy client JS، Lighthouse CI، a11y AA، وdark-only brand consistency.

## 🎯 نظرة عامة

المرحلة دي بتبني الواجهة العامة للمشروع: صفحة الـ landing الرئيسية، صفحات الميزات، الـ changelog، الـ blog، صفحات قانونية كاملة (Privacy / Terms / DPA / Cookies / Acceptable Use)، وكل الـ SEO infrastructure (sitemap، robots، structured data، OG images، Twitter cards). الهدف: الزائر الجديد يفهم في 5 ثواني إيه Zenith، ليه هو مختلف، وإن كل حاجة **مجانية بالكامل**.

**المستخدم/الزائر هيكسب إيه:**

- يفهم القيمة في ثواني من landing page بتركيز على "كل حاجة مجانية".
- يشوف الميزات الرئيسية مقسمة بشكل مرئي مع GIFs/videos توضيحية.
- يقرأ blog posts عن المنتج، الـ tips، الـ case studies.
- يطلع على changelog شفاف (كل release له page).
- يقرأ Privacy/Terms مكتوبة بلغة بشرية مفهومة بالعربي والإنجليزي.
- يتأكد إن بياناته محمية ومش هتتباع لأي حد.
- يقدر يحمل press kit + brand assets جاهزة.
- يشترك في newsletter اختياري بدون أي ضغط.
- يلاقي صفحات تليق بـ Open Graph + Twitter Card على السوشيال.
- يلاقي SEO ranking قوي على Google بفضل structured data + fast LCP.

**اللي هيتبني في المرحلة دي:**

- Landing page بـ hero + features grid + testimonials + FAQ + CTA.
- صفحات `/features/{feature}` لكل ميزة رئيسية (tasks, notes, habits, AI, etc.).
- `/changelog` بـ MDX-based posts (كل release له page منفصل).

- صفحات قانونية: `/legal/privacy`, `/legal/terms`, `/legal/dpa`, `/legal/cookies`, `/legal/acceptable-use`.
- صفحة `/security` بشرح كل الـ practices (encryption, RLS, audits).
- صفحة `/about` + `/team` (اختياري) + `/contact`.
- `/press` بـ brand assets (logo, screenshots, founder photos).
- Newsletter signup بـ double opt-in.
- Sitemap.xml + robots.txt + structured data (Organization, WebApp, BreadcrumbList).
- OG images dynamic بـ `@vercel/og`.
- Cookie banner بسيط (CookieConsent compliant).
- i18n من البداية: AR + EN.

```tsx
}
```

### 6) MDX blog + changelog

```tsx
// src/lib/blog/list.ts
import { promises as fs } from 'fs'
import path from 'path'
import matter from 'gray-matter'

const BLOG_DIR = path.join(process.cwd(), 'content/blog')

export interface BlogPost {
  slug: string
  title: string
  description: string
  date: string
  author: string
  category: 'product' | 'tips' | 'cases' | 'announcements'
  cover?: string
  content: string
}

export async function getAllBlogPosts(): Promise<BlogPost[]> {
  const files = await fs.readdir(BLOG_DIR)
  const posts = await Promise.all(
    files.filter(f => f.endsWith('.mdx')).map(async (file) => {
      const raw = await fs.readFile(path.join(BLOG_DIR, file), 'utf8')
      const { data, content } = matter(raw)
      return { slug: file.replace('.mdx',''), ...data, content } as BlogPost
    })
  )
  return posts.sort((a, b) => +new Date(b.date) - +new Date(a.date))
}

export async function getAllBlogSlugs() {
  return (await getAllBlogPosts()).map(p => p.slug)
}
```

```tsx
// src/app/blog/feed.xml/route.ts
import { getAllBlogPosts } from '@/lib/blog/list'

export async function GET() {
  const posts = await getAllBlogPosts()
  const items = posts.map(p => `
    <item>
      <title>${escapeXml(p.title)}</title>
      <link>https://zenithlifeos.app/blog/${p.slug}</link>
      <description>${escapeXml(p.description)}</description>
      <pubDate>${new Date(p.date).toUTCString()}</pubDate>
      <guid>https://zenithlifeos.app/blog/${p.slug}</guid>
    </item>
  `).join('')

  const xml = `<?xml version="1.0" encoding="UTF-8"?>
  <rss version="2.0">
    <channel>
      <title>Zenith Blog</title>
      <link>https://zenithlifeos.app/blog</link>
      <description>Articles about productivity, tools, and life management.</description>
      <language>ar</language>
      ${items}
    </channel>
  </rss>`
  return new Response(xml, { headers: { 'Content-Type': 'application/xml' } })
}

function escapeXml(s: string) {
  return s.replace(/[<>&'"]/g, c => ({ '<':'&lt;','>':'&gt;','&':'&amp;','\'':'&apos;','\"':'&quot;' })[c]!)
}
```

```tsx
'use client'
import { useEffect, useState } from 'react'

export function CookieBanner() {
  const [shown, setShown] = useState(false)
  useEffect(() => {
    if (!localStorage.getItem('cookie-consent')) setShown(true)
  }, [])
  if (!shown) return null
```

### 9) Newsletter signup (double opt-in)

```tsx
// src/app/api/newsletter/subscribe/route.ts
import 'server-only'
import { z } from 'zod'
import { serviceSupabase } from '@/lib/supabase/service'
import { sendEmail } from '@/lib/email/send'
import { randomBytes, createHash } from 'crypto'

const schema = z.object({ email: z.string().email() })

export async function POST(req: Request) {
  const { email } = schema.parse(await req.json())
  const token = randomBytes(32).toString('base64url')
  const hash  = createHash('sha256').update(token).digest('hex')

  await serviceSupabase.from('newsletter_pending').upsert({
    email, confirm_token_hash: hash,
    expires_at: new Date(Date.now() + 24*3600_000).toISOString(),
  })

  await sendEmail({
    to: email,
    subject: 'أكد اشتراكك في Zenith Newsletter',
    html: `
      <h2>أهلاً!</h2>
      <p>اضغط الرابط ده عشان تأكد اشتراكك:</p>
      <a href="https://zenithlifeos.app/api/newsletter/confirm?token=${token}">تأكيد الاشتراك</a>
      <p>الرابط صالح لـ 24 ساعة. لو مش انت اللي طلبت، تجاهل الإيميل.</p>
    `,
  })

  return Response.json({ ok: true, message: 'بعتنالك إيميل للتأكيد' })
}
```

```
# سياسة الخصوصية

## ما البيانات اللي بنجمعها؟
- بياناتك اللي بتنشأها في Zenith (مهام، ملاحظات، عادات، ...).
- بيانات الحساب (إيميل، اسم، صورة بروفايل اختياري).

## ما اللي مفيش بنعمله؟
- **مش بنبيع** بياناتك لأي حد أبداً.
- **مش بنشارك** محتواك مع third parties إلا لو إنت طلبت (integrations).
- **مش بنقرأ** محتوى Vault الخاص بيك أبداً (مشفر end-to-end).
- **مش بنستخدم** بياناتك لتدريب AI models.

## حقوقك (GDPR + CCPA)
- Right to access: نزّل كل بياناتك من /settings/export.
- Right to delete: احذف حسابك من /settings/account → كل البيانات هتنمسح خلال 30 يوم.
- Right to portability: export JSON/CSV/PDF متاح دائماً.
- Right to rectification: عدّل أي بيانات من /settings.

## كيف بنحمي بياناتك؟
- TLS 1.3 في كل transit.
- AES-256 encryption في الـ Vault.
- Row Level Security على كل DB tables.
- Audit logs لكل sensitive operations.
- Penetration testing سنوي.
```
