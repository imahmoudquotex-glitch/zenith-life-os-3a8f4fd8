### Sync Policy

- كل external item يخزن `external_id`, `provider`, `sync_version`.
- conflict policy موثقة: `remote_wins`, `local_wins`, أو `manual`.
- sync worker لا يكتب DB مباشرة؛ يستخدم canonical services فقط.

## 🧠 Master Architect Upgrade — Integrations Zero-Trust Gate

- **Token storage:** OAuth/access/refresh tokens encrypted server-side with envelope/key rotation، ولا تُخزن داخل Vault المستخدم ولا logs ولا webhooks ولا AI context.
- **OAuth hardening:** PKCE + state + nonce + exact redirect URI allowlist + scope minimization + revoke flow + refresh locking to prevent token race.
- **Webhook safety:** HTTPS only غير كافي؛ أضف SSRF/DNS rebinding/private IP block، HMAC with timestamp+nonce، replay window، body size limit، وtimeouts.
- **Telegram pairing:** pairing code one-time/10min، Telegram `update_id` idempotency، no password/email in bot، destructive commands confirmation.
- **Canonical data sync:** integrations لا تكتب tables legacy؛ تكتب عبر canonical services/system DBs مع `workspace_id` وconflict policy وaudit.

## 🎯 نظرة عامة

المرحلة دي بتربط Zenith بالعالم الخارجي: تكاملات OAuth جاهزة مع Google Calendar و Todoist و Notion و Slack، ونظام Webhooks خارج (Outgoing) عشان أي مستخدم يقدر يبني automation خاص بيه (Zapier / n8n / Make / Custom). كل token محمي بتشفير AES-GCM في الـ Vault، وكل webhook ممضي بـ HMAC-SHA256 ومحمي بـ retry queue + dead letter.

**المستخدم هيكسب إيه:**

- يربط حسابه على Google Calendar وأحداث الكالندر تتزامن أوتوماتيك في الاتجاهين.
- يربط Todoist وتاسكاته تتنقل لـ Zenith والعكس.
- يربط Slack ويتلقى إشعاراته على Zenith أو يدفع تحديثات لقناة معينة.
- يضيف Webhook URL خاص بيه ويستقبل event JSON موقّع كل ما يحصل حدث (task.created، habit.checkin، goal.completed، إلخ).
- لو الـ webhook URL وقع، Zenith بيعمل retry بـ exponential backoff (1 دقيقة → 5 → 15 → ساعة → 3 ساعات).
- بعد 5 محاولات فاشلة، الحدث يدخل dead letter queue ويتم تنبيه المستخدم.
- لو الـ integration health بقى مش كويس (3 errors متتالية)، تتعطل أوتوماتيك بإشعار للمستخدم.
- مفيش tokens تتسرّب في الـ logs ولا في الـ webhook payloads.

**اللي هيتبني في المرحلة دي:**

- جدول `integrations` بـ token encrypted (AES-GCM) + scope + health tracking.
- Token manager بيرفرش الـ token تلقائياً قبل انتهائه بـ 5 دقايق ويعمل atomic update.
- OAuth callbacks لكل provider (`/api/integrations/{provider}/callback`).
- Google Calendar two-way sync بـ cron كل 15 دقيقة (مش realtime عشان نقلل API quota).
- جدول `webhook_endpoints` بـ HTTPS only constraint + secret bcrypt-hashed + events array.
- جدول `webhook_deliveries` بـ retry queue + status enum (pending/sent/failed/dead) + next_retry timestamp.
- Dispatcher بيستبعد أي حقول vault من الـ payload (SAFE_FIELDS allowlist).
- HMAC-SHA256 signature في `X-Zenith-Signature` + nonce في `X-Zenith-Delivery`.
- BullMQ worker لمعالجة الـ retry queue بـ exponential backoff.
- Rate limit: 1000 outgoing webhooks/يوم لكل مستخدم.

### 3) Token encryption (AES-GCM via Vault key)

```tsx
// src/features/integrations/lib/token-crypto.ts
import 'server-only'
import { serverEnv } from '@/lib/env/server'

async function deriveKey(userId: string): Promise<CryptoKey> {
  const base = await crypto.subtle.importKey(
    'raw',
    new TextEncoder().encode(serverEnv.INTEGRATION_MASTER_KEY),
    { name: 'PBKDF2' }, false, ['deriveKey']
  )
  return crypto.subtle.deriveKey(
    { name: 'PBKDF2', salt: new TextEncoder().encode(userId),
      iterations: 100_000, hash: 'SHA-256' },
    base, { name: 'AES-GCM', length: 256 }, false, ['encrypt','decrypt']
  )
}

export async function encryptIntegrationToken(token: string, userId: string): Promise<string> {
  const key = await deriveKey(userId)
  const iv  = crypto.getRandomValues(new Uint8Array(12))
  const ct  = await crypto.subtle.encrypt({ name: 'AES-GCM', iv }, key,
    new TextEncoder().encode(token))
  const combined = new Uint8Array(iv.length + ct.byteLength)
  combined.set(iv); combined.set(new Uint8Array(ct), iv.length)
  return btoa(String.fromCharCode(...combined))
}

export async function decryptIntegrationToken(enc: string, userId: string): Promise<string> {
  const combined = new Uint8Array(atob(enc).split('').map(c => c.charCodeAt(0)))
  const iv  = combined.slice(0, 12)
  const ct  = combined.slice(12)
  const key = await deriveKey(userId)
  const pt  = await crypto.subtle.decrypt({ name: 'AES-GCM', iv }, key, ct)
  return new TextDecoder().decode(pt)
}
```

### 4) Token manager بـ auto-refresh

```tsx
// src/features/integrations/lib/token-manager.ts
import 'server-only'
import { serviceSupabase } from '@/lib/supabase/service'
import { encryptIntegrationToken, decryptIntegrationToken } from './token-crypto'
import { AppError } from '@/lib/result'
import { log } from '@/lib/monitoring/logger'

const REFRESH_URLS: Record<string,string> = {
  google_calendar: 'https://oauth2.googleapis.com/token',
  notion:          'https://api.notion.com/v1/oauth/token',
  todoist:         'https://todoist.com/oauth/access_token',
}

export async function getIntegrationToken(
  userId: string, provider: string
): Promise<string> {
  const { data } = await serviceSupabase.from('integrations')
    .select('access_token_enc, refresh_token_enc, token_expires_at, is_healthy')
    .eq('user_id', userId).eq('provider', provider).single()

  if (!data) throw new AppError('INTEGRATION_NOT_FOUND', provider, 404)
  if (!data.is_healthy) throw new AppError('INTEGRATION_UNHEALTHY', provider, 503)

  const accessToken = await decryptIntegrationToken(data.access_token_enc!, userId)

  // refresh لو فاضل أقل من 5 دقايق
  if (data.token_expires_at &&
      new Date(data.token_expires_at).getTime() - Date.now() < 5 * 60_000) {
    return refreshToken(userId, provider, data.refresh_token_enc!)
  }
  return accessToken
}

async function refreshToken(userId: string, provider: string, refreshEnc: string): Promise<string> {
  const refresh = await decryptIntegrationToken(refreshEnc, userId)
  const url     = REFRESH_URLS[provider]
  if (!url) throw new AppError('NO_REFRESH_URL', provider, 400)

  const res = await fetch(url, {
    method: 'POST',
    headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
    body: new URLSearchParams({
      grant_type:    'refresh_token',
      refresh_token: refresh,
      client_id:     process.env[`${provider.toUpperCase()}_CLIENT_ID`] ?? '',
      client_secret: process.env[`${provider.toUpperCase()}_CLIENT_SECRET`] ?? '',
    }),
  })
  if (!res.ok) {
    await serviceSupabase.rpc('record_integration_error', {
      p_user: userId, p_provider: provider, p_error: `refresh_failed:${res.status}`
    })
    throw new AppError('TOKEN_REFRESH_FAILED', provider, 502)
  }
  const fresh = await res.json() as { access_token: string; expires_in: number }
  const newEnc = await encryptIntegrationToken(fresh.access_token, userId)

  await serviceSupabase.from('integrations').update({
    access_token_enc: newEnc,
    token_expires_at: new Date(Date.now() + fresh.expires_in * 1000).toISOString(),
  }).eq('user_id', userId).eq('provider', provider)

  await serviceSupabase.rpc('record_integration_success', {
    p_user: userId, p_provider: provider
  })
  log.info('integration_token_refreshed', { userId, provider })
  return fresh.access_token
}
```

### 5) Google Calendar two-way sync

```tsx
// src/features/integrations/google-calendar/sync.ts
import 'server-only'
import { google } from 'googleapis'
import { serviceSupabase } from '@/lib/supabase/service'
import { getIntegrationToken } from '../lib/token-manager'
import { audit } from '@/lib/audit'
import { serverEnv } from '@/lib/env/server'

export async function syncGoogleCalendar(userId: string) {
  const accessToken = await getIntegrationToken(userId, 'google_calendar')
  const auth = new google.auth.OAuth2()
  auth.setCredentials({ access_token: accessToken })
  const calendar = google.calendar({ version: 'v3', auth })

  const timeMin = new Date(Date.now() - 30 * 86_400_000).toISOString()
  const timeMax = new Date(Date.now() + 90 * 86_400_000).toISOString()

  const { data } = await calendar.events.list({
    calendarId: 'primary', timeMin, timeMax,
    maxResults: 250, singleEvents: true, orderBy: 'startTime',
  })

  let synced = 0
  for (const ev of data.items ?? []) {
    if (!ev.summary || !ev.start) continue
    await serviceSupabase.from('calendar_events').upsert({
      user_id: userId,
      title:   ev.summary,
      start_at: ev.start.dateTime ?? `${ev.start.date}T00:00:00Z`,
      end_at:   ev.end?.dateTime  ?? `${ev.end?.date}T23:59:59Z`,
      timezone: ev.start.timeZone ?? 'UTC',
      all_day:  !ev.start.dateTime,
      metadata: { google_event_id: ev.id, source: 'google_calendar' },
    }, { onConflict: 'user_id,metadata->>google_event_id' })
    synced++
  }

  await serviceSupabase.rpc('record_integration_success', {
    p_user: userId, p_provider: 'google_calendar',
  })
  await audit({ actorId: userId, action: 'integration.google_cal.synced',
    metadata: { synced } })
  return { synced }
}
```

```sql
BEGIN
  IF NEW.attempt >= 5 AND NEW.status <> 'sent' THEN

  END IF;
  RETURN NEW;
```

```tsx
// src/lib/webhooks/dispatch.ts
import 'server-only'
import { createHmac } from 'crypto'
import { serviceSupabase } from '@/lib/supabase/service'
import { audit } from '@/lib/audit'

const SAFE_FIELDS: Record<string, string[]> = {
  'task.created':    ['id','title','priority','due_date','project_id'],
  'task.done':       ['id','title','completed_at'],
  'task.deleted':    ['id'],
  'habit.checkin':   ['id','habit_id','day_local'],
  'goal.completed':  ['id','title','completed_at'],
  'expense.created': ['id','amount_cents','category','date'],
  'review.done':     ['id','day_local','mood_score','energy_level'],
}

function sanitize(eventType: string, payload: Record<string, unknown>) {
  const allow = SAFE_FIELDS[eventType] ?? []
  return Object.fromEntries(
    Object.entries(payload).filter(([k]) => allow.includes(k))
  )
}

export async function dispatchWebhook(
  userId: string, eventType: string, payload: Record<string, unknown>
) {
  const { data: endpoints } = await serviceSupabase
    .from('webhook_endpoints')
    .select('id, url, secret, events')
    .eq('user_id', userId).eq('is_active', true)
    .contains('events', [eventType])
  if (!endpoints?.length) return

  const safePayload = sanitize(eventType, payload)

  for (const ep of endpoints) {
    const { data: delivery } = await serviceSupabase.from('webhook_deliveries')
      .insert({
        endpoint_id: ep.id, user_id: userId, event_type: eventType,
        payload: safePayload, status: 'pending', attempt: 0,
      }).select('id').single()
    // BullMQ job يلتقطها
    await enqueueDelivery(delivery!.id)
  }
  await audit({ actorId: userId, action: `webhook.queued.${eventType}` })
}
```

### 8) Sender بـ exponential backoff

```tsx
// src/features/webhooks/lib/sender.ts
import 'server-only'
import { createHmac } from 'crypto'
import { serviceSupabase } from '@/lib/supabase/service'
import { log } from '@/lib/monitoring/logger'

const RETRY_DELAYS = [60, 300, 900, 3600, 10800] // 1m, 5m, 15m, 1h, 3h (seconds)
const TIMEOUT_MS   = 10_000

export async function sendWebhook(deliveryId: string): Promise<void> {
  const { data: delivery } = await serviceSupabase
    .from('webhook_deliveries')
    .select('*, webhook_endpoints(url, secret)')
    .eq('id', deliveryId).single()
  if (!delivery || delivery.status === 'dead' || delivery.status === 'sent') return

  const endpoint = delivery.webhook_endpoints as { url: string; secret: string }
  const attempt  = delivery.attempt + 1
  const body     = JSON.stringify({
    event:    delivery.event_type,
    payload:  delivery.payload,
    delivery: deliveryId,
    attempt,
    ts:       Date.now(),
  })
  const sig = createHmac('sha256', endpoint.secret).update(body).digest('hex')

  try {
    const ctrl  = new AbortController()
    const timer = setTimeout(() => ctrl.abort(), TIMEOUT_MS)

    const res = await fetch(endpoint.url, {
      method: 'POST',
      headers: {
        'Content-Type':       'application/json',
        'X-Zenith-Signature': `sha256=${sig}`,
        'X-Zenith-Delivery':  deliveryId,
        'X-Zenith-Attempt':   String(attempt),
        'X-Zenith-Event':     delivery.event_type,
      },
      body, signal: ctrl.signal,
    })
    clearTimeout(timer)

    if (res.ok) {
      await serviceSupabase.from('webhook_deliveries').update({
        status: 'sent', attempt, sent_at: new Date().toISOString(),
        response_status: res.status,
      }).eq('id', deliveryId)
      log.info('webhook_sent', { deliveryId, attempt })
      return
    }
    throw new Error(`HTTP ${res.status}`)
  } catch (err) {
    const delaySec = RETRY_DELAYS[attempt - 1] ?? RETRY_DELAYS.at(-1)!
    const nextRetry = new Date(Date.now() + delaySec * 1000).toISOString()
    await serviceSupabase.from('webhook_deliveries').update({
      status:     attempt >= 5 ? 'dead' : 'failed',
      attempt,
      last_error: String(err),
      next_retry: attempt >= 5 ? null : nextRetry,
    }).eq('id', deliveryId)
    log.warn('webhook_delivery_failed', { deliveryId, attempt, error: String(err) })
  }
}
```

```tsx
// src/workers/webhook-retry.ts
import { Worker, QueueScheduler } from 'bullmq'
import { redisConn } from '@/lib/redis'
import { sendWebhook } from '@/features/webhooks/lib/sender'

new QueueScheduler('webhook-retry', { connection: redisConn })

export const webhookWorker = new Worker(
  'webhook-retry',
  async (job) => sendWebhook(job.data.deliveryId),
  { connection: redisConn, concurrency: 10 }
)

// cron يلتقط deliveries بـ status=pending OR (status=failed AND next_retry <= now())
export async function tickRetries() {
  const { data } = await serviceSupabase.from('webhook_deliveries')
    .select('id')
    .or('status.eq.pending,and(status.eq.failed,next_retry.lte.now())')
    .limit(100)
  for (const d of data ?? []) {
    await webhookQueue.add('send', { deliveryId: d.id }, { jobId: d.id })
  }
}
```
