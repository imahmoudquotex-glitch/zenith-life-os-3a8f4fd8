## 🧠 تطوير تنفيذي إضافي — Endpoint Security Wrapper Pack

كل route حساس يدخل من wrapper واحد: auth، tenant، capability، schema، CSRF/rate-limit، audit.

### Secure Mutation Wrapper

```tsx
export function secureMutation<I, O>(opts: {
  name: string
  schema: z.ZodSchema<I>
  capability: Capability
  csrf?: boolean
  rateLimit: { limit: number; windowSec: number }
  handler: (input: I, ctx: RequestContext) => Promise<O>
}) {
  return async (req: Request) => {
    const ctx = await requireRequestContext(req)
    await authorize(ctx, opts.capability)
    await rateLimit({ key: `${opts.name}:${ctx.userId}`, ...opts.rateLimit })
    if (opts.csrf) await verifyCsrfToken(req.headers.get('x-csrf-token'))
    const input = opts.schema.parse(await req.json())
    const result = await opts.handler(input, ctx)
    await audit.write({ action: opts.name, actorId: ctx.userId, workspaceId: ctx.workspaceId })
    return Response.json({ ok: true, data: result })
  }
}
```

### Security Gate

- zero critical/high findings قبل launch.
- secret scan على كل PR.
- admin 2FA mandatory.

```tsx
export async function safeFetchExternal(url: string, init?: RequestInit) {
  const first = await assertPublicHttpsUrl(url)
  const res = await fetch(first, { ...init, redirect: 'manual', signal: AbortSignal.timeout(5000) })
```

## 🧠 Master Architect Upgrade — Security Architecture Gate

- **Zero-trust baseline:** كل endpoint يثبت auth, tenant, capability, CSRF/rate limit عند الحاجة، وZod schema قبل أي side effect.
- **ZK/E2EE clarity:** استخدم Zero-Knowledge Encryption/E2EE للخزنة فقط. لا تدّعي ZKP إلا لو proof protocol فعلي موجود ومختبر.
- **Secrets lifecycle:** secrets inventory، rotation كل 90 يوم، break-glass procedure، وCI secret scanning إلزامي قبل merge.
- **SSRF everywhere:** webhooks, OG fetch, importers, clipper, AI tools، وimage proxies كلهم يستخدموا SSRF guard واحد مع DNS rebinding protection.
- **Security release gate:** لا launch قبل SAST/DAST/dependency scan/pen-test remediation: zero critical/high open findings.

## 🎯 نظرة عامة

المرحلة دي بتبني طبقات أمان دفاعية متعددة (defense in depth) على كل مستويات التطبيق: من الـ edge (CDN/WAF) لغاية الـ database. بنربط Cloudflare WAF أمام Vercel، بندخّل strict CSP، بنفعّل rate limiting على كل endpoint حساس، بنحمي من CSRF/XSS/SQLi/SSRF، بندير secrets بـ Vault rotation، بنلزم 2FA للـ admin، وبنجري security audits دورية. كل إجراء بيتسجل في audit log محصّن ضد التلاعب (append-only). حدود الاستخدام إدارية بحتة (ضد abuse) وليست أبداً تقييد للميزات المجانية.

**المستخدم هيكسب إيه:**

- ثقة تامة إن حسابه محمي بأعلى معايير.
- حماية من brute force attempts على تسجيل الدخول.
- 2FA تفعيل بدون تعقيد (TOTP + WebAuthn/Passkeys).
- سجل أمني لكل دخول (جهاز، IP، بلد، وقت).
- تنبيه فوري لو في دخول غريب.
- تسجيل خروج فوري من كل الأجهزة بضغطة واحدة.
- تصدير أو حذف بياناته (GDPR rights).
- توثيق واضح لكل access من admins.
- Vault encryption end-to-end للـ sensitive data.

**اللي هيتبني في المرحلة دي:**

- Cloudflare WAF rules + DDoS protection + bot mitigation.
- Content Security Policy (CSP) strict + nonce-based + report-only أولاً.
- Security headers كاملة (HSTS, X-Frame-Options, X-Content-Type, Permissions-Policy).
- Rate limiting metering بـ Redis sliding window + IP/user/endpoint scopes.
- CSRF tokens على كل state-changing requests.
- Input validation صارمة بـ Zod لكل endpoint.
- SSRF protection في أي outbound request (webhook, image fetch).
- SQL injection prevention (parameterized queries fully).
- 2FA: TOTP (otplib) + Passkeys (WebAuthn).
- Session management بـ rotation + device tracking + revoke-all.
- Login alerts (إيميل لو دخول من IP/جهاز جديد).
- Anomaly detection (دخول من بلد بعيد فجأة).
- Secrets rotation policy (every 90 days).
- Penetration testing سنوي + bug bounty program.
- SAST + DAST + dependency scanning في CI.
- Encrypted backups (مع المرحلة 32).

### 2) Rate limiting (Redis sliding window)

```tsx
// src/lib/rate-limit.ts
import 'server-only'
import { redis } from '@/lib/redis'
import { AppError } from '@/lib/result'

export async function rateLimit(opts: {
  key: string; limit: number; windowSec: number; burst?: number;
}) {
  const now = Date.now()
  const windowStart = now - opts.windowSec * 1000
  const pipe = redis.pipeline()
  pipe.zremrangebyscore(opts.key, 0, windowStart)
  pipe.zadd(opts.key, now, `${now}-${Math.random()}`)
  pipe.zcard(opts.key)
  pipe.expire(opts.key, opts.windowSec + 1)
  const results = await pipe.exec()
  const count = (results?.[2]?.[1] as number) ?? 0
  if (count > opts.limit) {
    throw new AppError('RATE_LIMITED', `${count}/${opts.limit} requests`, 429)
  }
  return { count, remaining: opts.limit - count }
}

// Usage:
// await rateLimit({ key: `login:${ip}`, limit: 5, windowSec: 60 })
// await rateLimit({ key: `signup:${ip}`, limit: 3, windowSec: 3600 })
// await rateLimit({ key: `ai:${userId}`, limit: 60, windowSec: 60 })
```

### 3) CSRF tokens

```tsx
// src/lib/security/csrf.ts
import 'server-only'
import { cookies } from 'next/headers'
import { randomBytes, createHmac, timingSafeEqual } from 'crypto'

const SECRET = process.env.CSRF_SECRET!

export async function issueCsrfToken(): Promise<string> {
  const raw = randomBytes(32).toString('base64url')
  const sig = createHmac('sha256', SECRET).update(raw).digest('base64url')
  const token = `${raw}.${sig}`
  ;(await cookies()).set('csrf', token, {
    httpOnly: false, secure: true, sameSite: 'lax', path: '/',
    maxAge: 86400,
  })
  return token
}

export async function verifyCsrfToken(headerToken: string | null) {
  if (!headerToken) throw new Error('CSRF_MISSING')
  const cookieToken = (await cookies()).get('csrf')?.value
  if (!cookieToken || cookieToken !== headerToken) throw new Error('CSRF_MISMATCH')
  const [raw, sig] = cookieToken.split('.')
  const expected = createHmac('sha256', SECRET).update(raw).digest('base64url')
  if (!timingSafeEqual(Buffer.from(sig), Buffer.from(expected))) throw new Error('CSRF_BAD_SIG')
}
```

### 4) SSRF protection

```tsx
// src/lib/security/ssrf-guard.ts
import { lookup } from 'dns/promises'

const PRIVATE_RANGES = [
  /^10\./, /^192\.168\./, /^172\.(1[6-9]|2\d|3[0-1])\./,
  /^127\./, /^169\.254\./, /^0\./, /^::1$/, /^fc00:/,
]

export async function assertPublicHttpsUrl(url: string) {
  const u = new URL(url)
  if (u.protocol !== 'https:') throw new Error('SSRF_NON_HTTPS')
  if (u.hostname === 'localhost') throw new Error('SSRF_LOCALHOST')
  const { address } = await lookup(u.hostname)
  if (PRIVATE_RANGES.some(r => r.test(address))) throw new Error('SSRF_PRIVATE_IP')
  return u
}
```

```tsx
// src/lib/security/totp.ts
import { authenticator } from 'otplib'
import { encrypt, decrypt } from '@/lib/security/vault'

authenticator.options = { window: 1, step: 30 }

export async function enableTotp(userId: string) {
  const secret = authenticator.generateSecret(32)
  const otpauth = authenticator.keyuri(userId, 'Zenith', secret)
  await serviceSupabase.from('user_totp').upsert({
    user_id: userId, secret_enc: await encrypt(secret),
  })
  return { secret, otpauth }
}

export async function verifyTotp(userId: string, code: string): Promise<boolean> {
  const { data } = await serviceSupabase.from('user_totp')
    .select('secret_enc').eq('user_id', userId).single()
  if (!data) return false
  const secret = await decrypt(data.secret_enc)
  return authenticator.verify({ token: code, secret })
}
```

### 7) Anomaly detection

```tsx
// src/lib/security/anomaly.ts
import 'server-only'
import { serviceSupabase } from '@/lib/supabase/service'
import { sendEmail } from '@/lib/email/send'

export async function checkLoginAnomaly(
  userId: string, ip: string, country: string, ua: string,
) {
  const { data: recent } = await serviceSupabase.from('user_sessions')
    .select('country_code, user_agent')
    .eq('user_id', userId)
    .order('last_seen', { ascending: false })
    .limit(20)

  const knownCountries = new Set(recent?.map(s => s.country_code) ?? [])
  const newCountry = !knownCountries.has(country)
  const tooFastTravel = await detectImpossibleTravel(userId, ip, country)

      to: await getEmailFor(userId),
```
