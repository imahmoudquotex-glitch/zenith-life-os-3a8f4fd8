# 🔥 مرحلة 31 — Infinite Canvas + Whiteboards

## 🧠 توسعة تنفيذية مميزة — Visual Thinking Operating System

مرحلة 31 ليست مجرد Whiteboard. هذه المرحلة هي **Visual Thinking Operating System** داخل المنتج: مساحة لا نهائية تجمع التفكير الحر، المخططات، الخرائط الذهنية، الرسومات، sticky notes، wireframes، diagrams، وربط كل عنصر بالـ pages/tasks/notes/goals/search/AI/realtime. الهدف إن المستخدم لا يخرج إلى Miro أو FigJam أو Excalidraw؛ كل التفكير البصري يحصل داخل المنصة نفسها، متصل بباقي المعرفة والعمل.

<aside>
🚀

ميزة قاتلة: الـ Canvas يجب أن يكون “صفحة تفكير حية” وليس مجرد رسم. كل شكل يمكن ربطه بصفحة، كل sticky note قابل للتحويل إلى task، كل frame قابل للتصدير، وكل رسم قابل للفهم بواسطة AI.

</aside>

### الفلسفة التصميمية

- **Canvas as database-aware space:** العناصر ليست pixels فقط؛ كل عنصر له ID، metadata، backlinks، comments، permissions، search text.
- **Infinite but structured:** المستخدم يقدر يرسم بحرية، لكن يقدر ينظم بـ frames, sections, layers, minimap, outlines.
- **Realtime first:** كل حركة أو تعديل shape يمر عبر Yjs/Hocuspocus من مرحلة 30.
- **AI-native:** AI يرى بنية اللوحة metadata/text/relationships وليس screenshot فقط.
- **Dark-mode masterpiece:** ألوان، grids، shadows، cursors، sticky notes كلها مصممة dark-only من البداية.
- **Desktop-first PWA:** اختصارات كيبورد قوية، mouse/trackpad gestures، multi-select، command palette.

## 🧭 Mental Model للـ AI/Claude Code

```
Canvas Page
  ↓
Tldraw Store / Y.Doc
  ↓
Canvas Elements: shapes, arrows, frames, notes, embeds
  ↓
Element Metadata: links, comments, tags, source entities
  ↓
Realtime Sync via Hocuspocus
  ↓
Persistence: binary CRDT state + searchable extracted text
  ↓
AI Canvas Understanding Layer
  ↓
Exports / Embeds / Backlinks / Automations
```

### قواعد تنفيذ صارمة

- لا تحفظ canvas كـ JSON ضخم داخل page content.
- لا تعتمد على screenshot للبحث أو الـ AI إلا كمدخل إضافي.
- لا ترسل Vault-linked canvas content إلى AI.
- لا تجعل export يجمّد الواجهة؛ استخدم worker/job للـ exports الثقيلة.
- لا تسمح لـ canvas embed بكسر صلاحيات الصفحة الأصلية.
- لا تجعل awareness/cursors persistent.
- لا تعمل paywall لأي tool أو export؛ الحدود فقط للأداء.

## 🔎 Metadata Extraction للبحث والـ AI

```tsx
export async function extractCanvasMetadata(args: {
  workspaceId: string
  canvasId: string
  snapshot: TLStoreSnapshot
}) {
  const rows = []
  for (const record of Object.values(args.snapshot.store)) {
    if (!isCanvasShape(record)) continue
    const text = extractTextFromShape(record)
    const links = extractEntityLinks(record)
    const bounds = extractBounds(record)
    rows.push({
      workspace_id: args.workspaceId,
      canvas_id: args.canvasId,
      element_id: record.id,
      kind: record.typeName,
      text_plain: text,
      linked_page_id: links.pageId ?? null,
      linked_task_id: links.taskId ?? null,
      linked_goal_id: links.goalId ?? null,
      tags: links.tags,
      ai_readable: !links.isSensitive,
      is_sensitive: links.isSensitive,
      bounds_json: bounds,
    })
  }
  await canvasMetadataRepo.replaceAll(args.canvasId, rows)
}
```

### Search behavior

- نصوص sticky/text/frames تدخل universal search خلال 30 ثانية.
- نتائج البحث تفتح canvas على العنصر المحدد، ليس الصفحة فقط.
- linked pages/tasks تظهر backlinks من canvas element.
- sensitive elements لا تدخل AI ولا search العام إلا حسب الصلاحيات.

```tsx
export async function buildCanvasAIContext(args: {
  workspaceId: string
  canvasId: string
  selectedElementIds?: string[]
}) {
  const elements = await canvasMetadataRepo.listAIReadable(args)
  assertNoVaultOrSecretElements(elements)
  return {
    canvasId: args.canvasId,
    elements: elements.map((e) => ({
      id: e.element_id,
      kind: e.kind,
      text: e.text_plain,
      links: {
        pageId: e.linked_page_id,
        taskId: e.linked_task_id,
        goalId: e.linked_goal_id,
      },
      bounds: e.bounds_json,
    })),
  }
}

export async function runCanvasAIJob(job: CanvasAIJob) {
  const context = await buildCanvasAIContext({
    workspaceId: job.workspaceId,
    canvasId: job.canvasId,
    selectedElementIds: job.inputElementIds,
  })

  return runAIWithQuota({
    workspaceId: job.workspaceId,
    userId: job.requestedBy,
    purpose: `canvas.${job.jobType}`,
    prompt: renderCanvasPrompt(job.jobType, context),
    dlp: { blockVault: true, blockSecrets: true, redactFinancialSensitive: true },
  })
}
```

- standalone canvas has own permission record.

```tsx
    hocuspocusUrl: `wss://realtime.zenith.app/canvas/${canvasId}`,
  })
  return <Tldraw store={store} autoFocus />
}
```
