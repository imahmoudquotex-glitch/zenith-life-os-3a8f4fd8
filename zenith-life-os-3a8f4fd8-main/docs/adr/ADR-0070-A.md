# ADR-0070-A: Zenith Repo Adoption as Design Baseline

**Status:** Accepted  
**Date:** 2026-05-16  
**Decision Makers:** Project Owner + Antigravity  

## Context

Wave 04 requires a design system, UI primitives, and auth UI. The Zenith repo (Lovable-generated) already provides:
- Full Dashboard with Life Score Ring, Energy Curve, XP/Level system
- Complete App Shell (Sidebar, Topbar, Search, Notifications)
- Design tokens in `tailwind.config.ts` + `styles.css`
- UI components (Button, Input, Badge, etc.) via shadcn/ui
- Dark-only theme with neon green accent
- RTL Arabic support with Cairo/Inter fonts

## Decision

**Adopt the Zenith repo as the single source of truth for visual design.** Do NOT create a parallel design system.

### Rules
1. All new pages use existing Zenith tokens and components
2. Auth/Settings/Landing pages match the established dark-neon aesthetic
3. No `packages/design-tokens` or `packages/ui-primitives` — tokens live in `styles.css`
4. Any new component must be a wrapper over existing Zenith/shadcn primitives
5. CSS variables defined once in `@theme inline` block, consumed via Tailwind utilities

## Consequences

### Positive
- Zero design drift — consistent visual language
- Faster development — no design system bootstrapping
- Lovable-quality Dashboard preserved as-is

### Negative
- Limited to shadcn/ui component API surface
- Custom components require matching the existing visual language manually

## Compliance

Verified by:
- `pnpm typecheck` → 0 errors ✅
- `pnpm build` → success ✅
- Visual consistency audit → all pages dark + neon green ✅
