# Accessibility (A11y) — Wave 04 Conventions

## Baseline: WCAG 2.1 AA

## Color Contrast

| Element | Minimum Ratio |
|---|---|
| Normal text | 4.5:1 |
| Large text (≥18px regular / ≥14px bold) | 3:1 |
| UI components & icons | 3:1 |

Current theme: dark background `#0a0a0a` + neon green accent — verified ≥ 4.5:1.

## Focus Management

- `focus-visible:ring-2 focus-visible:ring-accent` on every interactive element
- `outline: none` is **prohibited** without a replacement ring
- Return focus to trigger after Dialog/Sheet close

## Keyboard Navigation

- All interactive elements reachable via `Tab` in logical order
- `Enter` submits forms
- `Escape` closes dialogs/sheets
- Arrow keys navigate menus/tabs

## Screen Reader Support

- ARIA `role` attributes on custom components
- `aria-live="polite"` on toast regions
- `aria-busy` on loading states
- `aria-invalid` + `aria-describedby` on form error fields
- `aria-label` on icon-only buttons

## Reduced Motion

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

Already in `src/styles.css`.

## Skip Link

First focusable element in DOM: "تخطى للمحتوى الرئيسي"
Links to `#main-content` (or `#onboarding-main` on onboarding page).

## Heading Hierarchy

- One `<h1>` per page
- No skipped heading levels (h1 → h2 → h3)
- Never use headings for visual styling only

## Forms

- Every `<input>` has an associated `<label>` (visible or `sr-only`)
- Error summary at top of form + inline errors
- First errored field focused after failed submit
- `required` fields: `aria-required="true"` + visual asterisk
